Implementasi InteractWithCache Trait

Panduan manajemen memori sekunder (Redis/Memcached) menggunakan pola *Group Index Tracking* di Ajocell v2.

Cara Kerja Group Index Tracking

Ketika kamu menyimpan cache, trait otomatis mencatat penandanya (*key*) ke sebuah array indeks kelompok. Saat *clear group* dipanggil, hanya isi kelompok itu yang dihancurkan dari RAM.

// 📂 Struktur Index Pelacak di RAM "ajocell:index:categories" => [ "ajocell:categories:prabayar", "ajocell:categories:pascabayar" ];
// 🔥 Efek Menjalankan clearCacheGroup('categories') ✔ Menghapus key prabayar dari RAM (Flush)
✔ Menghapus key pascabayar dari RAM (Flush)
ℹ Cache "ajocell:products:telkomsel" tetap AMAN & UTUH!

Pemicuan pembersihan kelompok cache agar data di sisi klien tetap akurat.

notFoundResponse();

        // Ini memastikan request GET berikutnya akan dipaksa mengambil data terbaru dari database

    /**
     * DELETE /api/v2/admin/categories/{id}
     */
    public function destroy($id): JsonResponse
    {
        $category = Category::find($id);
        if (!$category) return $this->notFoundResponse();

        $category->delete();

        // 🎯 Hancurkan cache agar kategori yang terhapus tidak muncul lagi di API pengguna
        $this->clearCacheGroup('categories');

        return $this->successResponse('Kategori berhasil dihapus permanen & cache dibersihkan.');
    }
}
