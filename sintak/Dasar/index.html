Kembali ke Dashboard API & Logic Cheat Sheet

Komparasi Back-End vs Front-End

Referensi manajemen data dari sisi server (PHP/Laravel/SQL) hingga penanganan di sisi klien (JS/React).

1

Definisi Data & Struktur Array

Bagaimana data diinisialisasi di backend dan dikelola di frontend.

PHP (Assoc Array) 
$user = [
    "nama" => "Johar",
    "role" => "Admin"
];
Laravel (API Response) 
return response()->json([
    'status' => 'success',
    'data' => $user
], 200);
JavaScript (Object) 
const user = {
    nama: "Johar",
    role: "Admin"
};
React (State Fetch) 
const [user, setUser] = 
  useState(null);
// Diisi via Axios/Fetch API
2

Pengkondisian & Validasi (If / Else)

Validasi keamanan di server berbanding kondisi rendering di browser.

PHP (Standard If) 
if (!$token) {
    die("Unauthorized");
}
Laravel (Abort/Validation) 
if ($validator->fails()) {
    abort(422, "Data Invalid");
}
// Atau Menggunakan Gate::allows()
JavaScript 
if (!isLoggedIn) {
    router.push('/login');
}
React (Guard Component) 
if (!user) return ;

return ;
3

Pemrosesan Data Massal (Looping & Map)

Manipulasi koleksi data di server vs mapping komponen di sisi client.

PHP (Foreach) 
foreach ($products as $p) {
    $prices[] = $p['price'];
}
Laravel (Collections) 
$prices = collect($products)
    ->map(fn($p) => $p->price)
    ->filter(fn($price) => $price > 0);
JavaScript (.map / .filter) 
const prices = products
  .map(p => p.price)
  .filter(p => p > 0);
React (Render List) 
return (
  prices.map((price, i) => (
    <span key={i}>{price}</span>
  ))
);

Hubungan SQL Query & API Layer

Bagaimana database menyediakan data mentah untuk dibungkus oleh Back-End API.

Database Architecture

SQL Native Query (Relasi & Filter) 

-- Mengambil produk berdasarkan relasi vendor & cek status aktif
SELECT p.id, p.name, v.vendor_name, p.price 
FROM products p
LEFT JOIN vendors v ON p.vendor_id = v.id
WHERE p.status = 'active'
ORDER BY p.id DESC;

Representasi Backend (Laravel Eloquent) 

// Query Builder / Eloquent equivalents yang dieksekusi di Controller API
$products = Product::with('vendor')
    ->where('status', 'active')
    ->latest()
    ->get();

Struktur Aliran Data: Search & Pagination (API to React TS)

Implementasi pencarian dan paginasi data Guru dari Laravel API hingga di-mapping di React TypeScript.

End-to-End Architecture
Laravel Backend GuruController.php

Menangani *request* pencarian nama/NIP dan mengembalikan pagination standard JSON.

public function index(Request $request)
{
    $query = Guru::query();

    // Fitur Pencarian Dinamis
    if ($request->has('search')) {
        $search = $request->search;
        $query->where('nama', 'like', "%{$search}%")
              ->orWhere('nip', 'like', "%{$search}%");
    }

    // Paginasi otomatis (Contoh: 10 data per halaman)
    $teachers = $query->paginate(10);

    return response()->json([
        'status' => 'success',
        'message' => 'Data Guru Berhasil Diambil',
        'meta' => [
            'current_page' => $teachers->currentPage(),
            'last_page' => $teachers->lastPage(),
            'per_page' => $teachers->perPage(),
            'total' => $teachers->total(),
        ],
        'data' => $teachers->items() // Array objek Guru
    ], 200);
}
React TypeScript TeacherList.tsx

Mendefinisikan *Interface TS* untuk validasi tipe data, melakukan fetch, dan melakukan *mapping* ke UI.

import React, { useState, useEffect } from 'react';

// 1. Definisi Tipe Data (TypeScript Interface)
interface Teacher {
  id: number;
  nama: string;
  nip: string;
  mapel: string;
}

interface ApiMeta {
  current_page: number;
  last_page: number;
  total: number;
}

export const TeacherList: React.FC = () => {
  const [teachers, setTeachers] = useState<Teacher[]>([]);
  const [meta, setMeta] = useState<ApiMeta | null>(null);
  const [search, setSearch] = useState<string>('');
  const [page, setPage] = useState<number>(1);

  // 2. Fetch Data dari Laravel API
  useEffect(() => {
    const fetchTeachers = async () => {
      const res = await fetch(`/api/guru?search=${search}&page=${page}`);
      const json = await res.json();
      if (json.status === 'success') {
        setTeachers(json.data);
        setMeta(json.meta);
      }
    };
    fetchTeachers();
  }, [search, page]); // Triger ulang jika search atau page berubah

  return (
    <div>
      {/* Input Search */}
      <input value={search} onChange={(e) => setSearch(e.target.value)} />

      {/* 3. Mapping Data dengan TypeScript */}
      <ul>
        {teachers.map((teacher: Teacher) => (
          <li key={teacher.id}>
            {teacher.nama} - NIP: {teacher.nip} ({teacher.mapel})
          </li>
        ))}
      </ul>

      {/* Tombol Paginasi */}
      <button onClick={() => setPage(p => Math.max(p - 1, 1))}>Prev</button>
      <button onClick={() => setPage(p => p + 1)}>Next</button>
    </div>
  );
};
