Implementasi ErrorExcep Wrapper

Panduan membungkus *Business Logic* menggunakan metode penanganan pengecualian terpusat Ajocell v2.

Arsitektur Eksekusi Aman

Metode safeExecuteWithResponse bertindak sebagai interceptor. Daripada menulis block try-catch yang berulang di setiap fungsi, kamu cukup melemparkan logika utama ke dalam bentuk klausa penutup (*anonymous function / callback*).

1. Gate Security Check Otomatis memvalidasi hak akses user via Spatie/Laravel Gate.
2. Callback Execution Mengeksekusi logika utama database, mutasi saldo, atau hit vendor.
3. Polymorphic Catch Memilah otomatis HTTP status code tanpa crash SQL mentah.

Penerapan pada TransaksiController.php

Kasus nyata: Pembelian Pulsa / Token Listrik PPOB H2H

Controller Level 
safeExecuteWithResponse(
            'create-transaksi', // Kemampuan di App\Policies atau Gate
            Transaksi::class,   // Target Model pengamanan
            function () use ($request) {
                
                // 1. Validasi input form secara ketat
                $validated = $request->validate([
                    'produk_sku'   => 'required|string|exists:products,sku',
                    'nomor_tujuan' => 'required|numeric|digits_between:10,14',
                ]);

                // 2. Simulasi Aturan Bisnis Tingkat Lanjut (Contoh: Pengecekan Saldo)
                $user = auth()->user();
                $hargaProduk = 50000; // Contoh statis dari database master

                if ($user->saldo < $hargaProduk) {
                    // Dilempar ke AppException (Otomatis ditangkap status 422 oleh core engine)
                    throw new AppException('Transaksi ditolak! Saldo akun Anda tidak mencukupi untuk melakukan transaksi ini.', 422);
                }

                // 3. Logic database utama (Aman dari QueryException SQL Inject / Konflik Data)
                $transaksi = Transaksi::create([
                    'user_id' => $user->id,
                    'sku'     => $validated['produk_sku'],
                    'target'  => $validated['nomor_tujuan'],
                    'status'  => 'pending'
                ]);

                // Nilai return ini otomatis masuk ke variable $result dalam wrapper dan berstatus 201 Created
                return $transaksi;
            },
            'Gagal memproses transaksi PPOB. Sistem mendeteksi kegagalan interkoneksi.' // Custom Error fallback
        );
    }
}

Analisis Proteksi Cyber Security & Exception Mapping

Mencegah Information Disclosure

Saat database mengalami error (misal nama kolom typo atau database down), QueryException akan ditangkap. Pesan asli kueri dibuang ke log internal (logger()->error()) dan user luar hanya menerima pesan general "Terjadi konflik data pada database Core". Ini menutup celah penyerang menganalisis struktur tabel database melalui pesan error sistem.

Sentralisasi Validasi & Otorisasi

Fungsi Gate::authorize($ability, $model) dijalankan di awal block wrapper sebelum baris kueri mana pun diproses. Jika token tidak memiliki izin, program langsung melempar AuthorizationException dan memicu $this->forbiddenResponse() (HTTP 403), menghemat penggunaan memori server.
