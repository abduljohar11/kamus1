Configuration

"Mengatur bagaimana aplikasi Anda berperilaku di berbagai lingkungan."

Analogi Dunia Nyata

Bayangkan aplikasi Anda adalah sebuah Smartphone. Config adalah menu "Settings". Di sana Anda mengatur bahasa, kecerahan, dan koneksi Wi-Fi. Sedangkan .env adalah "Kartu SIM". Isinya data unik seperti nomor telepon dan paket data yang berbeda-beda tergantung kartu (lingkungan) yang dimasukkan, tanpa mengubah settingan asli HP-nya.

1. Lokasi File Config

Semua file konfigurasi untuk Laravel disimpan di dalam direktori config. Setiap file mengembalikan array berisi opsi pengaturan. 

// Contoh mengambil nilai config di Controller/Blade
$name = config('app.name');

// Memberikan nilai default jika tidak ada
$timezone = config('app.timezone', 'UTC');

2. Environment Configuration (.env)

Sangat disarankan untuk menyimpan data sensitif (seperti password database) di file .env agar tidak ikut terunggah ke Git. 

# File: .env
APP_NAME=KamusCoding
APP_ENV=local
APP_KEY=base64:randomkey
APP_DEBUG=true
APP_URL=http://localhost

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=my_db
DB_USERNAME=root
DB_PASSWORD=secret

Mengakses Nilai .env

Gunakan helper env() di dalam file config untuk menarik data dari .env. 

// Di dalam config/database.php
'connections' => [
    'mysql' => [
        'driver' => 'mysql',
        'host' => env('DB_HOST', '127.0.0.1'),
        'database' => env('DB_DATABASE', 'forge'),
        // ...
    ],
],

3. Configuration Caching

Untuk meningkatkan performa di Production, gabungkan semua file config menjadi satu file cache cepat.

Peringatan: Jika Anda menjalankan perintah cache, fungsi env() hanya akan mengembalikan nilai jika dipanggil dari dalam file config. Jangan gunakan env() langsung di Controller! 
# Membuat cache config
php artisan config:cache

# Menghapus cache config
php artisan config:clear

4. Maintenance Mode

Gunakan fitur ini saat Anda ingin melakukan perbaikan pada aplikasi sehingga user mendapatkan halaman "Sedang Perbaikan". 

# Aktifkan maintenance mode
php artisan down --refresh=15 --retry=60

# Nonaktifkan (Aplikasi jalan lagi)
php artisan up
