Dashboard Eloquent Relationships

Menghubungkan Data

Panduan lengkap memahami bagaimana tabel saling berinteraksi di Laravel, mulai dari One-to-One hingga Polymorphic.

1

One to One (Satu ke Satu)

Analogi: KTP

Satu Penduduk hanya memiliki satu KTP. Sebaliknya, satu KTP hanya milik satu Penduduk. Hubungannya sangat eksklusif.

Model A: User | Model B: Phone

Model User.php (Induk) 

public function phone()
{
    // Saya memiliki satu Phone
    return $this->hasOne(Phone::class);
}

Model Phone.php (Anak) 

public function user()
{
    // Saya milik User
    return $this->belongsTo(User::class);
}
2

One to Many (Satu ke Banyak)

Analogi: Artikel Blog

Satu Artikel Post bisa memiliki banyak Komentar. Tapi, satu Komentar pasti hanya menempel pada satu Artikel Post saja. 

// Model Post.php
public function comments()
{
    return $this->hasMany(Comment::class);
}

// Model Comment.php
public function post()
{
    return $this->belongsTo(Post::class);
}

Cara Mengambil Data: 

$post = Post::find(1);

// Loop semua komentar milik post ini
foreach ($post->comments as $comment) {
    echo $comment->message;
}
3

Many to Many (Banyak ke Banyak)

Analogi: Ekstrakurikuler

Satu Siswa bisa ikut banyak Ekskul (Basket, Musik). Satu Ekskul juga punya banyak Siswa.

Wajib: Butuh tabel perantara (Pivot Table), misal: ekskul_siswa. 

// Model Siswa.php
public function ekskuls()
{
    // Parameter 2: nama tabel pivot (jika tidak standar)
    return $this->belongsToMany(Ekskul::class, 'ekskul_siswa');
}

// Model Ekskul.php
public function siswas()
{
    return $this->belongsToMany(Siswa::class, 'ekskul_siswa');
}

Attach & Detach (Menambah/Menghapus Relasi): 

$siswa = Siswa::find(1);

// Menambahkan siswa ke ekskul ID 5
$siswa->ekskuls()->attach(5);

// Menghapus siswa dari ekskul ID 5
$siswa->ekskuls()->detach(5);

// Sinkronisasi (Hapus yang lama, simpan yang baru di array ini)
$siswa->ekskuls()->sync([1, 2, 5]);
4

Has Many Through

Analogi: Kakek - Ayah - Cucu

Negara punya banyak Provinsi. Provinsi punya banyak Kota.
Jika Negara ingin mengambil data semua Kota secara langsung (melewati Provinsi), gunakan relasi ini. 

// Model Negara.php
public function cities()
{
    // Ambil data City, melewati model Province
    return $this->hasManyThrough(City::class, Province::class);
}
5

Polymorphic (Si Serba Bisa)

Analogi: Komentar Media Sosial

Di Facebook, Anda bisa komen di Status, bisa juga komen di Foto, atau komen di Video. Daripada membuat tabel status_comments, photo_comments, dll, kita buat satu tabel Comments yang bisa menempel ke model apa saja.

Struktur Database (Migration): 

$table->id();
$table->text('body');
// Ini akan membuat 2 kolom: commentable_id & commentable_type
$table->morphs('commentable'); 
$table->timestamps();

Model Implementation: 

// Model Comment.php (Anak)
public function commentable()
{
    // Saya bisa jadi milik siapa saja (Post, Video, dll)
    return $this->morphTo();
}

// Model Post.php (Induk 1)
public function comments()
{
    return $this->morphMany(Comment::class, 'commentable');
}

// Model Video.php (Induk 2)
public function comments()
{
    return $this->morphMany(Comment::class, 'commentable');
}

Eager Loading (N+1 Problem)

Wajib dipahami agar aplikasi tidak lemot.

Masalah N+1

Bayangkan Anda disuruh membeli 10 buku. Cara Salah (Lazy): Anda pergi ke toko, beli buku A, pulang. Pergi lagi ke toko, beli buku B, pulang. (10x bolak-balik).
Cara Benar (Eager): Anda pergi sekali, beli semua buku A-J, lalu pulang. (1x jalan).

Jangan Lakukan Ini (Lazy): 

// Query 1x
$books = Book::all();

foreach ($books as $book) {
    // Query 100x (jika ada 100 buku)
    // SELECT * FROM authors WHERE id = ?
    echo $book->author->name;
}

Lakukan Ini (Eager): 

// Gunakan with()
// SELECT * FROM books
// SELECT * FROM authors WHERE id IN (1,2,3...)
$books = Book::with('author')->get();

foreach ($books as $book) {
    echo $book->author->name;
}

6

Fitur Lanjutan Relationship (Processing Data)

Setelah berhasil menghubungkan tabel, kita butuh cara efisien untuk mengolah, menyaring, dan mengambil data tambahan dari hubungan tersebut.

A. Extra Columns di Tabel Pivot

Analogi: Catatan di Buku Absen

Jika tabel Pivot adalah Buku Absen Ekskul, terkadang kita tidak cuma mencatat "Siapa ikut apa", tapi juga "Kapan dia bergabung" atau "Apa jabatannya". Data tambahan ini disimpan langsung di tabel pivot. 

// Di Model Siswa.php
public function ekskuls() {
    // Memberitahu Laravel bahwa ada kolom 'jabatan' di tabel pivot
    return $this->belongsToMany(Ekskul::class)->withPivot('jabatan')->withTimestamps();
}

// Cara Menampilkan di Blade/Controller
foreach ($siswa->ekskuls as $ekskul) {
    echo $ekskul->nama_ekskul;
    echo $ekskul->pivot->jabatan; // Mengakses kolom tambahan
}

B. Menghitung Jumlah Relasi (withCount)

Analogi: Counter di Rak Buku

Daripada Anda menghitung satu per satu buku di rak (yang memakan waktu), Anda cukup melihat Label Angka di depan rak yang bertuliskan: "Rak ini berisi 50 buku". Sangat cepat dan efisien. 

// Di Controller: Ambil Post beserta jumlah komentarnya
$posts = Post::withCount('comments')->get();

// Di Blade: Otomatis muncul variabel {nama_relasi}_count
@foreach($posts as $post)
    <div>
        <h2>{{ $post->title }}</h2>
        <span>Jumlah Komentar: {{ $post->comments_count }}</span>
    </div>
@endforeach

C. Menyaring Berdasarkan Anak (whereHas)

Analogi: Filter Orang Tua

Anda ingin mencari Daftar Orang Tua yang memiliki Anak dengan Nilai A. Sistem tidak akan menampilkan semua orang tua, melainkan hanya mereka yang anaknya memenuhi kriteria tersebut. 

// Mencari User yang punya Postingan dengan kata kunci 'Laravel'
                $users = User::whereHas('posts', function($query) {
                    $query->where('title', 'like', '%Laravel%');
                })->get();
                // Kebalikannya: Mencari User yang TIDAK punya postingan sama sekali
                $usersTanpaPost = User::doesntHave('posts')->get();
