Membangun UserService: Logika CRUD & Kueri Kompleks

Sesi 1: Memisahkan core logic bisnis dari Controller ke dalam berkas Service khusus, mengimplementasikan pencarian dinamis, pagination, serta aksi manipulasi data.

ALUR & RASIOMALISASI

Mengapa menggunakan Service Layer? Di proyek skala menengah-atas, Controller seharusnya hanya bertugas menerima HTTP request dan mengembalikan JSON response (Thin Controller). Segala bentuk kueri database yang kompleks, filter pencarian, hashing, dan manipulasi objek data diletakkan di kelas Service.

HTTP Request ➔ Controller ➔ UserService (Logika & Query) ➔ DB Model ➔ Response JSON

Tahap 1 Inisialisasi File UserService

Laravel tidak menyediakan command native untuk membuat Service. Buatlah berkas PHP baru secara manual di dalam direktori baru: app/Services/UserService.php.

Sintaks Penuh: app/Services/UserService.php 
<?php

namespace App\Services;

use App\Models\User;
use Illuminate\Contracts\Pagination\LengthAwarePaginator;
use Illuminate\Support4\Facades\Hash;

class UserService
{
    /**
     * Mengambil data user dengan pencarian dinamis, filter role, dan pagination.
     * Logika: Jika parameter 'search' atau 'role' dikirim, filter kueri Eloquent secara kondisional.
     */
    public function getAllPaginated(array $filters, int $perPage = 10): LengthAwarePaginator
    {
        $query = User::query();

        // Logika Pencarian: Cari berdasarkan nama, email, atau nomor induk (NIP/NISN)
        if (!empty($filters['search'])) {
            $search = $filters['search'];
            $query->where(function ($q) use ($search) {
                $q->where('name', 'like', "%{$search}%")
                  ->orWhere('email', 'like', "%{$search}%")
                  ->orWhere('nomor_induk', 'like', "%{$search}%");
            });
        }

        // Logika Filter Role: Memisahkan data Guru atau Murid secara spesifik
        if (!empty($filters['role']) && in_array($filters['role'], ['guru', 'murid'])) {
            $query->where('role', $filters['role']);
        }

        // Mengembalikan object paginator berurutan dari data terbaru
        return $query->latest()->paginate($perPage);
    }

    /**
     * Menyimpan data user baru ke database (Create Action)
     */
    public function create(array $data): User
    {
        // Password di-hash secara otomatis jika menggunakan casting di Model, 
        // namun eksplisit hashing di service menjamin keamanan di layer bisnis.
        if (isset($data['password'])) {
            $data['password'] = Hash::make($data['password']);
        }

        return User::create($data);
    }

    /**
     * Memperbarui data entitas user berdasarkan ULID (Update Action)
     */
    public function update(string $id, array $data): User
    {
        $user = User::findOrFail($id);

        if (!empty($data['password'])) {
            $data['password'] = Hash::make($data['password']);
        } else {
            // Hapus key password dari array jika user tidak berniat mengganti password
            unset($data['password']);
        }

        $user->update($data);

        return $user;
    }

    /**
     * Menghapus baris data user berdasarkan ULID dari database (Delete Action)
     */
    public function delete(string $id): bool
    {
        $user = User::findOrFail($id);
        
        return $user->delete();
    }
}

Analisis Kode & Parameter Sintaks

$query->where(function ($q) ...)

Pengelompokan kueri (*Query Grouping*) sangat krusial saat menggabungkan where dan orWhere. Jika tidak dikelompokkan dalam fungsi penutup closure, logika filter role di luar bisa rusak akibat interferensi kondisi OR.

LengthAwarePaginator

Return type contract yang memastikan output fungsi berupa objek pagination standar Laravel yang memuat metadata link halaman, total baris, halaman aktif saat ini, hingga parameter pencarian sebelumnya.

unset($data['password'])

Logika protektif pada fungsi update(). Jika client mengirim data modifikasi profile namun mengosongkan kolom password, kita membuang key tersebut agar password lama di DB tidak tertimpa string kosong atau hash string kosong.
