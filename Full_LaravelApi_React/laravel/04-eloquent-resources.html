Eloquent Resources & Aturan Validasi Mutasi Data

Bagaimana mengamankan payload input pendaftaran/perubahan data Guru & Murid, lalu memformat ulang struktur JSON response-nya agar konsisten dikonsumsi oleh React Frontend.

LOGIKA DATA FLOW (INPUT ➔ OUTPUT)

Arsitektur API yang matang wajib memisahkan validasi data masuk dan standardisasi data keluar. Form Request memastikan payload dari client bersih dan valid. Setelah diproses database, Eloquent Resource bertindak sebagai presenter layer yang bertugas menyembunyikan kolom sensitif DB (seperti password) dan menyatukan format data keluar.

INPUT LAYER

1. Form Request: Validasi Mutasi Data

Berikut adalah aturan validasi untuk memastikan data Guru (NIP) dan Murid (NISN) dikirim dengan benar, termasuk penanganan *ignore rule* berbasis ULID saat operasi update profil dilakukan.

Aksi Create: app/Http/Requests/StoreUserRequest.php 
<?php

namespace App\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Validation\Rule;

class StoreUserRequest extends FormRequest
{
    public function authorize(): bool
    {
        return true;
    }

    public function rules(): array
    {
        return [
            'name'        => ['required', 'string', 'max:255'],
            'email'       => ['required', 'string', 'email', 'max:255', 'unique:users,email'],
            'password'    => ['required', 'string', 'min:8'],
            'role'        => ['required', 'string', Rule::in(['guru', 'murid'])],
            
            // Validasi Nomor Induk: Wajib diisi dan tidak boleh kembar di DB
            'nomor_induk' => ['required', 'string', 'max:50', 'unique:users,nomor_induk'],
        ];
    }
}
Aksi Update: app/Http/Requests/UpdateUserRequest.php 
<?php

namespace App\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Validation\Rule;

class UpdateUserRequest extends FormRequest
{
    public function authorize(): bool
    {
        return true;
    }

    public function rules(): array
    {
        // Menangkap ULID user dari segmen route URL api/users/{user}
        $userId = $this->route('user');

        return [
            'name'        => ['sometimes', 'required', 'string', 'max:255'],
            
            // Mengabaikan pengecekan unik jika email/nomor induk adalah milik user ini sendiri
            'email'       => ['sometimes', 'required', 'string', 'email', 'max:255', Rule::unique('users', 'email')->ignore($userId)],
            'role'        => ['sometimes', 'required', 'string', Rule::in(['guru', 'murid'])],
            'nomor_induk' => ['sometimes', 'required', 'string', 'max:50', Rule::unique('users', 'nomor_induk')->ignore($userId)],
            
            // Password nullable: jika form dikosongkan frontend, password lama tidak berubah
            'password'    => ['nullable', 'string', 'min:8'],
        ];
    }
}
OUTPUT LAYER

2. Eloquent Resource: UserResource

Mengubah data mentah hasil query menjadi payload JSON siap pakai untuk React. Di sini kita membuat logika visual penamaan label identitas secara otomatis berdasarkan peran (*role*) user.

File: app/Http/Resources/UserResource.php 
<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class UserResource extends JsonResource
{
    /**
     * Transformasikan data model menjadi array JSON.
     */
    public function toArray(Request $request): array
    {
        $isGuru = $this->role === 'guru';

        return [
            'uid'             => $this->id, // Meng-expose string ULID
            'nama_lengkap'    => $this->name,
            'email_aktif'     => $this->email,
            'peran_user'      => strtoupper($this->role), // Mengubah 'guru' -> 'GURU'
            
            // Pemetaan Seragam untuk Frontend React
            'nomor_identitas' => $this->nomor_induk,
            'tipe_identitas'  => $isGuru ? 'NIP (Nomor Induk Pegawai)' : 'NISN (Nomor Induk Siswa Nasional)',
            
            'terdaftar_pada'  => $this->created_at->toIso8601String(),
        ];
    }
}

3. Implementasi Gabungan di UserController

Lihat bagaimana data mengalir masuk melalui Form Request, diproses, lalu keluar dengan aman dibungkus oleh Resource Transformer. 

// Panggil di baris atas file controller Anda:
// use App\Http\Requests\StoreUserRequest;
// use App\Http\Requests\UpdateUserRequest;
// use App\Http\Resources\UserResource;

/**
 * Aksi Simpan User Baru (Store)
 */
public function store(StoreUserRequest $request): JsonResponse
{
    // 1. Data lolos sensor Form Request diambil via $request->validated()
    // 2. Dilempar ke Service layer untuk diinsert ke database
    $user = $this->userService->create($request->validated());

    // 3. Output dikembalikan dengan bungkus UserResource tunggal
    return response()->json([
        'success' => true,
        'message' => 'Akun baru berhasil di-indeks ke sistem',
        'data'    => new UserResource($user)
    ], 201); // 201: HTTP Created
}

/**
 * Aksi Perbarui Profile User (Update)
 */
public function update(UpdateUserRequest $request, string $id): JsonResponse
{
    // 1. Validasi otomatis mengecek unique email & nomor induk dengan mengabaikan ID ini
    // 2. Update via Service Layer
    $user = $this->userService->update($id, $request->validated());

    // 3. Response JSON terformat kembali ke Frontend React
    return response()->json([
        'success' => true,
        'message' => 'Data profil berhasil diperbarui',
        'data'    => new UserResource($user)
    ], 200);
}

Analisis Proteksi & Alur Data

$request->validated() vs $request->all()

Selalu gunakan validated(). Jika hacker mencoba menyuntikkan payload tambahan secara ilegal (seperti kolom is_admin = true), Laravel akan membuang key tersebut karena tidak terdaftar secara resmi di dalam rules Form Request.

Rule::unique()->ignore($userId)

Sangat krusial untuk mencegah kegagalan sistem saat update. Tanpa adanya fungsi pengecualian ini, jika seorang Guru melakukan pembaruan nama profil tanpa mengubah NIP-nya, database akan menolak dan memicu error karena nomor induk tersebut sudah terpakai (oleh dirinya sendiri).
