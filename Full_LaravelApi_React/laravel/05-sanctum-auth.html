Autentikasi Headless: Laravel Sanctum & Fortify

Panduan integrasi backend mandiri (SPA/API) menggunakan kombinasi token Laravel Sanctum dan mesin autentikasi Laravel Fortify dengan modifikasi login via Username.

LOGIKA ARSITEKTUR AUTH

Dalam arsitektur *headless* API (React Frontend + Laravel Backend), kita memadukan dua paket dengan tugas berbeda:

  • Laravel Fortify: Menangani logika mesin utama backend (proses pengecekan kredensial, *throttling/rate limiting*, enkripsi).
  • Laravel Sanctum: Menangani penerbitan token (*Stateful Token* atau *Bearer Token*) setelah Fortify berhasil memverifikasi kredensial user.
React Payload (username + password) ➔ Fortify (Kredensial Matcher) ➔ Sanctum (Issue Bearer Token) ➔ JSON Response

01 Instalasi Composer Packages

Secara bawaan Laravel modern sudah menyertakan Sanctum. Kita hanya perlu menambahkan Fortify untuk mengelola subsistem autentikasinya secara *headless*.

Terminal CLI
# 1. Install Fortify via Composer
composer require laravel/fortify
# 2. Publish berkas konfigurasi bawaan Fortify
php artisan fortify:install

02 Ubah Identitas Login Menjadi Username/Nomor Induk

Buka file konfigurasi config/fortify.php. Cari kata kunci 'username' dan ganti dari default 'email' menjadi nama kolom database target Anda (misalnya 'nomor_induk' untuk mengakomodasi NIP/NISN).

File: config/fortify.php 
<?php

return [
    // ... konfigurasi lainnya
    
    // Logika: Mengubah kolom deteksi autentikasi primer dari email ke nomor_induk
    'username' => 'nomor_induk',

    // Menandakan guard default yang digunakan
    'guard' => 'web',

    // Mematikan rendering view HTML bawaan Fortify karena kita menggunakan Headless API
    'views' => false,

    // ... fitur bawaan yang ingin diaktifkan (pastikan hanya mengaktifkan 'login')
    'features' => [
        Features::login(),
    ],
];

03 Paksa Balasan Selalu Berupa JSON (Bukan Redirect 302)

Fortify secara default akan melempar *302 Redirect* kembali ke halaman login jika mendeteksi kegagalan otentikasi. Kita wajib mencegat perilaku tersebut di dalam app/Providers/FortifyServiceProvider.php menggunakan penyesuaian kustom.

File: app/Providers/FortifyServiceProvider.php 
<?php

namespace App\Providers;

use App\Models\User;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Hash;
use Illuminate\Support\ServiceProvider;
use Laravel\Fortify\Fortify;

class FortifyServiceProvider extends ServiceProvider
{
    public function register(): void
    {
        //
    }

    public function boot(): void
    {
        // 1. Logika Kustom Proses Pencocokan Kredensial Login
        Fortify::authenticateUsing(function (Request $request) {
            // Validasi input data masuk
            $request->validate([
                'nomor_induk' => 'required|string',
                'password'    => 'required|string',
            ]);

            // Cari user berdasarkan nomor_induk di database
            $user = User::where('nomor_induk', $request->nomor_induk)->first();

            // Verifikasi password kecocokan hash
            if ($user && Hash::check($request->password, $user->password)) {
                return $user;
            }

            // Logika: Jika gagal, return null agar Fortify tahu kredensial tidak valid
            return null;
        });

        // 2. Cegat Respons Sukses Fortify Agar Mengembalikan Token Sanctum & JSON
        $this->app->singleton(
            \Laravel\Fortify\Http\Responses\LoginResponse::class,
            new class implements \Laravel\Fortify\Contracts\LoginResponse {
                public function toResponse($request) {
                    $user = auth()->user();
                    
                    // Terbitkan token bearer Sanctum saat berhasil masuk
                    $token = $user->createToken('api-device-token')->plainTextToken;

                    return response()->json([
                        'success' => true,
                        'message' => 'Autentikasi berhasil, token diterbitkan.',
                        'token'   => $token,
                        'role'    => $user->role
                    ], 200);
                }
            }
        );

        // 3. Cegat Respons Gagal Validasi Fortify Agar Berformat JSON Bersih (422)
        $this->app->singleton(
            \Laravel\Fortify\Http\Responses\FailedVerificationResponse::class,
            new class implements \Laravel\Fortify\Contracts\FailedVerificationResponse {
                public function toResponse($request) {
                    return response()->json([
                        'success' => false,
                        'message' => 'Nomor induk atau sandi salah, akses ditolak.'
                    ], 422);
                }
            }
        );
    }
}

04 Mengunci Endpoint Menggunakan Token Sanctum

Buka file rute API Anda (routes/api.php). Bungkus endpoint yang membutuhkan pengecekan login dengan middleware auth:sanctum.

File: routes/api.php 
<?php

use App\Http\Controllers\Api\UserController;
use Illuminate\Support\Facades\Route;

// Endpoint publik (Bisa diakses siapa saja tanpa token)
Route::post('/login', [Laravel\Fortify\Http\Controllers\AuthenticatedSessionController::class, 'store']);

// Endpoint terproteksi (Wajib menyertakan header 'Authorization: Bearer {token}')
Route::middleware('auth:sanctum')->group(function () {
    
    // Contoh mengambil profil user yang sedang aktif login
    Route::get('/user-profile', function (Request $request) {
        return response()->json([
            'success' => true,
            'data'    => $request->user()
        ]);
    });

    // CRUD User yang aman karena terkunci di dalam group middleware
    Route::apiResource('users', UserController::class);
});

Analisis Mekanisme Autentikasi API

$user->createToken()->plainTextToken

Fungsi utama Sanctum untuk menjatuhkan string token acak terenkripsi ke database. String *plain text* inilah yang wajib ditangkap oleh React dan disimpan ke dalam *LocalStorage/Cookies* untuk dikirim kembali pada request berikutnya.

$this->app->singleton() Override

Teknik *Inversion of Control* (IoC). Kita mendaftarkan class kustom kita ke dalam wadah aplikasi Laravel untuk menimpa class bawaan Fortify, sehingga respons yang semula berupa pengalihan halaman web (302) dipaksa berubah total menjadi struktur objek data JSON.
