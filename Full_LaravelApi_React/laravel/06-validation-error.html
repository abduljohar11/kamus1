Penerapan ApiResponder & EnumExtractor Trait

Panduan taktis implementasi struktur helper global menggunakan PHP Traits untuk menyeragamkan JSON payload dan mengamankan pemrosesan Backed Enum pada Controller.

MENGAPA MENGGUNAKAN PHP TRAITS?

Trait adalah mekanisme reusabilitas kode pada rumpun pemrograman OOP PHP yang memungkinkan sebuah class (seperti Controller atau Service) mengadopsi kumpulan method kustom secara langsung tanpa perlu melakukan pewarisan kelas induk (inheritance). Ini menjaga kode Anda tetap mematuhi prinsip DRY (Don't Repeat Yourself).

TRAIT 1

Mekanisme Kerja & Contoh ApiResponder

Trait ApiResponder membungkus fungsi enkapsulasi data ke dalam format JSON yang seragam. Method inti jsonResponse() otomatis memisahkan data sukses, pesan kesalahan validasi (HTTP 422), hingga kegagalan server (HTTP 500) agar React Frontend dapat mendeteksi status dengan konsisten.

Implementasi di app/Http/Controllers/Api/ProductController.php 
<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Traits\ApiResponder;
use Illuminate\Http\JsonResponse;

class ProductController extends Controller
{
    // Menggunakan trait di dalam class
    use ApiResponder;

    /**
     * Contoh 1: Respons Sukses Pembuatan Data (201 Created)
     */
    public function store(): JsonResponse
    {
        $product = ['id' => '01J6G...', 'name' => 'Pulsa Telkomsel 10K', 'price' => 11200];

        // Memanggil langsung method dari ApiResponder
        return $this->createdResponse('Produk prabayar berhasil ditambahkan.', $product);
    }

    /**
     * Contoh 2: Penanganan Gagal Validasi Form Input (422 Fail)
     */
    public function manualValidationCheck(): JsonResponse
    {
        $errors = [
            'phone_number' => ['Format nomor telepon tidak valid.'],
            'provider'     => ['Provider tidak ditemukan di vendor mapping.']
        ];

        // Memanggil format khusus validation response pembawa list array merah
        return $this->validationResponse($errors, 'Silakan periksa kembali form pengisian Anda.');
    }
}

TRAIT 2

Mekanisme Kerja & Contoh EnumExtractor

Saat menggunakan Fitur AsBackedEnum pada Model Eloquent Laravel, kolom database akan dibungkus menjadi sebuah Objek Enum PHP. Mencoba mencetak objek tersebut secara langsung sebagai string (echo $user->role) akan memicu Fatal Crash Error. Trait EnumExtractor memotong rantai error tersebut dengan mengekstrak nilai string murninya secara aman.

Implementasi Aman Pada Logic Layer Backend 
<?php

namespace App\Services;

use App\Models\User;
use App\Traits\EnumExtractor;

class OrderProcessingService
{
    // Inject Trait pengekstrak nilai enum ke dalam service
    use EnumExtractor;

    public function processTransaction(User $user, array $payload)
    {
        /**
         * MASALAH: Kolom $user->role mengembalikan objek casted enum (UserRole::GURU)
         * AMAN: extractEnumValue() mendeteksi objek tersebut dan mengembalikan string murni 'guru'
         */
        $roleString = $this->extractEnumValue($user, 'role'); 

        if ($roleString === 'guru') {
            // Berikan diskon khusus guru sekolah pada transaksi PPOB / Buku
            $payload['price'] = $payload['price'] - 500;
        }

        // Contoh Ekstraksi langsung dari Instance Enum tanpa lewat Model
        // Mengembalikan string murni 'pulsa' dari BackedEnum objek CategoryType::PULSA
        $categoryString = $this->extractEnumValue($payload['category_enum']);
        
        return $payload;
    }
}
Mengapa Trait Ini "Kebal 100%" Crash?

Perhatikan baris sensor utama pada kode trait: $target instanceof BackedEnum. Melalui pengecekan tipe data runtime ini, sistem tahu apakah variabel tersebut berupa string biasa atau objek Enum berjenis Backed (memiliki properti ->value). Jika terdeteksi sebagai Enum, properti nilainya diambil; jika data string biasa, fungsi langsung melakukan casting aman tanpa interupsi error.
