UserController & Registrasi Route API

Sesi 3: Menyatukan lapisan validasi input dan logika bisnis ke dalam controller, serta membuka gerbang endpoint RESTful API melalui perutean Laravel.

SINKRONISASI AKHIR LAYER aplikasi

Pada tahap ini, UserController bertindak murni sebagai konduktor lalu lintas data. Ia menangkap input yang telah lolos sensor dari Form Request, melemparkannya ke UserService untuk dieksekusi ke DB, lalu membungkus outputnya dalam standarisasi response JSON HTTP Status Code (200 OK, 201 Created, atau 204 No Content).

1 UserController (Thin Controller)

Gunakan Dependency Injection pada constructor untuk memuat instalasi UserService secara otomatis. Simpan di app/Http/Controllers/Api/UserController.php.

Sintaks Penuh: app/Http/Controllers/Api/UserController.php 
<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Http\Requests\StoreUserRequest;
use App\Http\Requests\UpdateUserRequest;
use App\Services\UserService;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;

class UserController extends Controller
{
    protected UserService $userService;

    // Inject UserService ke dalam controller
    public function __construct(UserService $userService)
    {
        $this->userService = $userService;
    }

    /**
     * Menampilkan daftar user (Index) + Fitur Search & Pagination
     */
    public function index(Request $request): JsonResponse
    {
        // Menangkap filter parameter '?search=' dan '?role=' dari URL query string
        $filters = $request->only(['search', 'role']);
        $users = $this->userService->getAllPaginated($filters, 10);

        return response()->json([
            'success' => true,
            'message' => 'Daftar data user berhasil diambil',
            'data'    => $users
        ], 200);
    }

    /**
     * Membuat data user baru (Store)
     */
    public function store(StoreUserRequest $request): JsonResponse
    {
        // Data yang diambil hanya data yang sudah tervalidasi oleh StoreUserRequest
        $user = $this->userService->create($request->validated());

        return response()->json([
            'success' => true,
            'message' => 'User baru berhasil ditambahkan',
            'data'    => $user
        ], 201); // 201: HTTP Created
    }

    /**
     * Memperbarui entitas data user (Update)
     */
    public function update(UpdateUserRequest $request, string $id): JsonResponse
    {
        $user = $this->userService->update($id, $request->validated());

        return response()->json([
            'success' => true,
            'message' => 'Data user berhasil diperbarui',
            'data'    => $user
        ], 200);
    }

    /**
     * Menghapus baris data user dari sistem (Destroy)
     */
    public function destroy(string $id): JsonResponse
    {
        $this->userService->delete($id);

        return response()->json([
            'success' => true,
            'message' => 'User berhasil dihapus dari database'
        ], 200);
    }
}

2 Konfigurasi API Routing

Buka file routes/api.php. Gunakan metode pembungkus Route::apiResource untuk mendaftarkan semua endpoint CRUD sekaligus (Index, Store, Update, Destroy) tanpa menyertakan route View Form (create/edit) yang tidak diperlukan untuk API headless.

Sintaks Penuh: routes/api.php 
<?php

use App\Http\Controllers\Api\UserController;
use Illuminate\Support\Facades\Route;

/*
|--------------------------------------------------------------------------
| API Routes
|--------------------------------------------------------------------------
*/

// Mendaftarkan endpoint CRUD User secara otomatis dengan prefix '/api/users'
Route::apiResource('users', UserController::class);

// Hasil pemetaan endpoint otomatis yang terbentuk di latar belakang:
// GET    /api/users          ➔ index()   (Ambil Semua User + Paging)
// POST   /api/users          ➔ store()   (Tambah User Baru)
// PUT    /api/users/{user}   ➔ update()  (Ubah Data User Berdasarkan ULID)
// DELETE /api/users/{user}   ➔ destroy() (Hapus User Berdasarkan ULID)

Analisis Alur Eksekusi RESTful

$request->validated()

Berbeda dengan $request->all(), fungsi ini menjamin hanya parameter data yang memiliki aturan validasi resmi di dalam file Form Request saja yang akan lolos ke database. Data asing kiriman hacker (seperti manipulasi paksa kolom isAdmin atau role lewat payload mentah) akan otomatis terbuang.

Route::apiResource()

Mempersingkat deklarasi rute kodingan Anda. Perintah ini mengisolasi perutean agar bersih dari rute create dan edit yang biasanya memicu rendering halaman HTML blade tradisional.
