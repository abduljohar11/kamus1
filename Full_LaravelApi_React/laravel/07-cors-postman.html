Panduan Integrasi & Pengujian

HTTP Methods Cheat Sheet & Postman Integration

Panduan komprehensif struktur request API. Memahami apa yang harus dikirim pada metode GET, POST, PUT, dan PATCH lengkap dengan visualisasi payload JSON standar industri.

HEADER WAJIB PADA POSTMAN (PENTING!)

Sebelum menembak endpoint apa pun di Postman (kecuali login publik), Anda wajib menyertakan komponen Headers berikut agar tidak terkena error CORS, batasan hak akses, atau dialihkan ke halaman login web biasa:

Accept: application/json (Memaksa Laravel selalu merespons format JSON murni)
Content-Type: application/json (Wajib ada jika mengirim data Body/Payload)
Authorization: Bearer {token_sanctum_anda} (Kunci masuk rute terproteksi)
GET

Membaca Data (Read)

Idempotent

Digunakan untuk mengambil informasi dari server. Metode GET tidak boleh memiliki Request Body/Payload JSON. Parameter penyaringan atau pencarian wajib diletakkan pada segmen URL (Query String).

Contoh Struktur URL di Postman:
GET https://api.ajocell.com/api/users?role=guru&search=ahmad&per_page=15
POST

Membuat Data Baru (Create)

Non-Idempotent

Digunakan untuk mengirimkan payload data baru ke server untuk disimpan ke dalam database. Data dikirim melalui tab Body ➔ raw ➔ JSON di Postman.

Contoh Payload JSON (Pendaftaran Guru Baru): 
{
    "name": "Arif Rahman, S.Pd",
    "email": "arif.rahman@sekolah.sch.id",
    "password": "sandiRahasia123",
    "role": "guru",
    "nomor_induk": "199208152018021004"
}
PUT

Mengganti Data Total (Replace)

Idempotent

Digunakan untuk memperbarui data secara menyeluruh. Filosofi PUT adalah mengganti seluruh objek lama dengan objek baru yang dikirim. Jika ada kolom wajib yang tidak Anda kirimkan, database berpotensi mengosongkan atau me-reset kolom tersebut menjadi nilai default.

URL Target: PUT /api/users/01J6G9XB...
Contoh Payload JSON (Wajib Mengirimkan Seluruh Kolom Properti): 
{
    "name": "Arif Rahman Hakim, M.Pd", 
    "email": "arif.rahman@sekolah.sch.id",
    "role": "guru",
    "nomor_induk": "199208152018021004"
}
PATCH

Mengubah Data Parsial (Modify)

Non-Idempotent

Digunakan untuk melakukan modifikasi data secara sebagian (parsial). Anda hanya perlu mengirimkan properti kolom yang ingin Anda ubah saja tanpa mengganggu nilai kolom lainnya di database (Sangat cocok untuk fitur ubah password atau ganti nama profil).

URL Target: PATCH /api/users/01J6G9XB...
Contoh Payload JSON (Hanya Mengubah Password Saja): 
{
    "password": "SandiBaruGuru2026"
}

STUDI KASUS NYATA: PAYLOAD TRANSAKSI PPOB (AJOCELL)

Sebagai gambaran arsitektur tingkat lanjut pada sistem PPOB h2h (Host-to-Host) Ajocell v2, berikut adalah bentuk kueri payload yang dikirim menggunakan metode **POST** saat agen melakukan pembelian token listrik pintar atau pulsa reguler:

POST /api/v2/transactions/purchase
{
    "master_product_sku": "PLN20",          // SKU tunggal internal Ajocell
    "customer_target_id": "53102488912",    // Nomor HP atau ID Pelanggan PLN
    "client_reference_id": "TRX-88231-XYZ", // ID unik transaksi dari sisi aplikasi client
    "payment_method": "SALDO_KEAGENAN"
}
