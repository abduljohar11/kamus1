Separation of Concerns: Struktur Folder React TypeScript

Panduan menyusun arsitektur folder berskala besar menggunakan TypeScript. Memisahkan UI, logika bisnis, dan integrasi API dengan jaminan kontrak data (*Type Safety*) yang ketat.

MENGAPA WAJIB TYPESCRIPT DI SCALABLE FRONTEND?

Dengan bermigrasi dari `.js/.jsx` ke `.ts/.tsx`, kita mendapatkan proteksi penuh saat melakukan integrasi API:

  • Mencegah Undefined Error: TypeScript mendeteksi eror saat compile jika komponen mengakses properti objek backend yang typo atau tidak eksis.
  • Kontrak Data API (Types/Interfaces): Struktur respons JSON dari backend Laravel dipetakan menjadi tipe data statis di frontend.
  • Autocompletion Sempurna: IDE/VS Code otomatis memberikan saran properti data saat mengetik kode.

Cetak Biru Pohon Direktori TypeScript src/

Perhatikan perubahan ekstensi file menjadi **`.ts`** untuk file logika/murni kode, dan **`.tsx`** untuk file yang mengandung elemen UI (JSX):

TYPES CONTRACT

1. Mapping Model Data (`src/types/user.ts`)

Mendefinisikan tipe data statis agar React mengetahui secara pasti properti apa saja yang dimiliki oleh objek guru hasil kiriman backend. 

// Definisi tipe data kaku untuk model User
export interface User {
  uid: string;
  nama_lengkap: string;
  email: string;
  role: 'admin' | 'guru' | 'developer';
  nomor_identitas: string;
}

// Interface global yang menyelaraskan format ApiResponder dari Laravel
export interface ApiResponse<T> {
  status: 'success' | 'fail' | 'error';
  message: string;
  data: T; // Menggunakan Generics agar tipe data dalam 'data' bisa dinamis (User, Product, dll)
  errors?: Record<string, string[]>;
}
LAYER API

2. Data Layer: Implementasi Return Type (`src/api/userApi.ts`)

Mengikat fungsi HTTP Request menggunakan contract type. Fungsi di bawah dijamin mengembalikan data bertipe ApiResponse penampung array User. 

import axiosInstance from './axiosInstance';
import { User, ApiResponse } from '../types/user';

export const userApi = {
  // Promise<ApiResponse<User[]>> memastikan output data adalah array dari object User sesuai type contract
  getAllUsers: async (role: string = ''): Promise<ApiResponse<User[]>> => {
    const response = await axiosInstance.get<ApiResponse<User[]>>(`/api/users?role=${role}`);
    return response.data;
  },

  createUser: async (payload: Omit<User, 'uid'>): Promise<ApiResponse<User>> => {
    const response = await axiosInstance.post<ApiResponse<User>>('/api/users', payload);
    return response.data;
  }
};
LAYER LOGIC

3. Logic Layer: Type State Guard (`src/pages/guru/useGuruLogic.ts`)

Inisialisasi state useState dikunci menggunakan type data array User agar tidak bisa diisi oleh sembarang tipe data lain. 

import { useState, useEffect } from 'react';
import { userApi } from '../../api/userApi';
import { User } from '../../types/user';

export function useGuruLogic() {
  // Mengunci tipe data array User pada state list
  const [guruList, setGuruList] = useState<User[]>([]);
  const [isLoading, setIsLoading] = useState<boolean>(false);
  const [errorMessages, setErrorMessages] = useState<string | null>(null);

  const fetchGuru = async (): Promise<void> => {
    setIsLoading(true);
    try {
      const result = await userApi.getAllUsers('guru');
      setGuruList(result.data); // Valid karena result.data bertipe User[]
    } catch (err) {
      setErrorMessages('Gagal memuat data guru dari server.');
    } finally {
      setIsLoading(false);
    }
  };

  useEffect(() => {
    fetchGuru();
  }, []);

  return { guruList, isLoading, errorMessages, refreshData: fetchGuru };
}
LAYER VIEW

4. View Layer: Autocompletion JSX (`src/pages/guru/GuruList.tsx`)

Saat menuliskan fungsi looping guru.nama_lengkap, text editor Anda akan secara otomatis mengenali properti tersebut dan melindunginya dari kesalahan pengetikan huruf kapital. 

import React from 'react';
import { useGuruLogic } from './useGuruLogic';
import { Button } from '@/components/ui/button';

export default function GuruList(): React.JSX.Element {
  // Destructuring state dengan inferensi tipe data otomatis dari custom hook
  const { guruList, isLoading, errorMessages } = useGuruLogic();

  if (isLoading) return <div className="text-slate-500 text-sm">Sedang menyinkronkan data...</div>;
  if (errorMessages) return <div className="text-red-500 text-sm">{errorMessages}</div>;

  return (
    <div className="space-y-4">
      <div className="flex justify-between items-center">
        <h1 className="text-xl font-bold">Daftar Manajemen Guru (TSX v4)</h1>
        <Button variant="default">Tambah Rekor Guru</Button>
      </div>

      <ul className="divide-y divide-slate-100">
        {guruList.map((guru) => (
          <li key={guru.uid} className="py-3 flex justify-between text-sm">
            {/* Kebal Eror Typo karena properti dikawal ketat oleh Interface User */}
            <span className="font-medium text-slate-900">{guru.nama_lengkap}</span>
            <span className="text-slate-500 font-mono">{guru.nomor_identitas}</span>
          </li>
        ))}
      </ul>
    </div>
  );
}
ALUR DATA

5. Anatomi JSON Backend vs Interface Frontend

Ketika API Backend (misal dari controller Laravel) mengirimkan format JSON standar, TypeScript akan memetakan skema objek tersebut melalui keyword interface agar dikenali oleh sistem frontend.

Muntahan JSON dari API Backend: 
{
  "status": "success",
  "message": "Data berhasil dimuat",
  "data": {
    "uid": "01J6G9XB...",
    "nama_lengkap": "Abdul Johar, S.Kom",
    "email": "abdul.johar@sekolah.sch.id",
    "role": "guru",
    "nomor_identitas": "199402112026031001"
  }
}
Ditangkap TypeScript Interface: 
// src/types/user.ts
export interface User {
  uid: string;
  nama_lengkap: string;
  email: string;
  role: 'admin' | 'guru' | 'developer';
  nomor_identitas: string;
}

export interface ApiResponse<T> {
  status: string;
  message: string;
  data: T; // Generics mengikat objek User
}
AXIOS SETUP

6. Konfigurasi Penembak API (`src/api/axiosInstance.ts`)

Gunakan interceptor untuk menyuntikkan token Bearer secara otomatis ke dalam komponen *headers* sebelum request terbang ke backend server. 

import axios, { AxiosInstance, InternalAxiosRequestConfig } from 'axios';

const axiosInstance: AxiosInstance = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'https://api.ajocell.com',
  headers: {
    'Accept': 'application/json',
    'Content-Type': 'application/json',
  },
  timeout: 10000, // Otomatis putus jika server macet lebih dari 10 detik
});

// Interceptor menyuntikkan Token keamanan dari LocalStorage secara dinamis
axiosInstance.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    const token = localStorage.getItem('ajocell_token');
    if (token && config.headers) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);

export default axiosInstance;
COMPONENTS .TSX

7. Konsumsi Data Aman Pada File View (`src/pages/guru/GuruList.tsx`)

Ini adalah implementasi penuh di dalam berkas UI. Menggunakan custom hook yang mengembalikan state beralias tipe data kaku. Proteksi ini mematikan celah kesalahan pemanggilan variabel data. 

import React from 'react';
import { useGuruLogic } from './useGuruLogic';
import { Button } from '@/components/ui/button';

export default function GuruList(): React.JSX.Element {
  // TypeScript otomatis mendeteksi bahwa guruList bertipe User[]
  const { guruList, isLoading, errorMessages, refreshData } = useGuruLogic();

  return (
    <div class="p-6 bg-white rounded-xl shadow-sm border border-slate-100">
      <div class="flex justify-between items-center mb-6">
        <h2 class="text-xl font-bold text-slate-800">Sistem Buku Guru - Data Staff</h2>
        <Button onClick={refreshData} variant="outline" size="sm" disabled={isLoading}>
          {isLoading ? 'Menyinkronkan...' : 'Refresh Data'}
        </Button>
      </div>

      {errorMessages && (
        <div class="p-3 bg-red-50 border border-red-200 text-red-600 rounded-lg text-sm mb-4">
          {errorMessages}
        </div>
      )}

      <div class="overflow-x-auto">
        <table class="w-full text-left text-sm border-collapse">
          <thead class="bg-slate-50 text-slate-400 font-semibold uppercase text-xs">
            <tr>
              <th class="p-3">Nama Lengkap</th>
              <th class="p-3">Identitas (NUPTK)</th>
              <th class="p-3">Hak Akses Akun</th>
            </tr>
          </thead>
          <tbody class="divide-y divide-slate-100 text-slate-700">
            {guruList.length === 0 && !isLoading ? (
              <tr><td colSpan={3} class="p-4 text-center text-slate-400">Kosong. Tidak ada data guru.</td></tr>
            ) : (
              guruList.map((guru) => (
                <tr key={guru.uid} class="hover:bg-slate-50/80 transition">
                  {/* IDE Anda akan memberikan autocompletion untuk nama_lengkap, nomor_identitas, & role */}
                  <td class="p-3 font-medium text-slate-900">{guru.nama_lengkap}</td>
                  <td class="p-3 font-mono text-xs">{guru.nomor_identitas}</td>
                  <td class="p-3">
                    <span class="px-2 py-0.5 bg-blue-50 text-blue-600 rounded text-xs font-semibold capitalize">
                      {guru.role}
                    </span>
                  </td>
                </tr>
              ))
            )}
          </tbody>
        </table>
      </div>
    </div>
  );
}

GOLDEN RULES UNTUK ARSITEKTUR TYPESCRIPT FRONTEND

  • Jangan Pernah Menggunakan tipe any: Menggunakan any sama saja mematikan fungsi pengawasan utama dari TypeScript.
  • Gunakan Omit / Pick Utility: Saat mengirim payload data baru di mana properti uid belum dibuat oleh backend database, manfaatkan Omit<User, 'uid'> untuk membuang properti tersebut secara aman dari validasi.
  • Selalu Sinkronkan Enum: Jika di database Laravel Anda menggunakan tipe kolom enum (seperti kategori data type prefix PPOB Ajocell atau Role User), pastikan tipe string literal di berkas .ts frontend disamakan persis (Case-Sensitive).
