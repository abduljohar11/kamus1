Data Mutation & Validation

Advanced Axios: CUD Operations & Async Validation Handling

Menguasai siklus mutasi data (POST/PUT/DELETE) menggunakan useMutation TanStack Query, menangkap error validasi field dari backend, dan strategi otomatisasi penyegaran state UI.

PERBEDAAN QUERY VS MUTATION

Jika useQuery digunakan murni untuk **membaca data (GET)** secara otomatis saat komponen di-mount, maka useMutation digunakan untuk **mengubah/memanipulasi data (POST/PUT/DELETE)** yang dipicu oleh aksi manual user (seperti klik tombol submit form).

01 Ekstensi API Layer Untuk Mutasi (`src/api/userApi.ts`)

Kita tambahkan fungsi penembak payload data baru dan penghapusan data. Gunakan utility Omit untuk membuang properti id otomatis dari type contract jika diperlukan. 

import axiosInstance from './axiosInstance';
import { User, ApiResponse } from '../types/user';

export const userApi = {
  // ... fungsi GET dari modul sebelumnya
  
  // POST: Membuat rekor baru
  createUser: async (payload: Omit<User, 'uid'>): Promise<ApiResponse<User>> => {
    const response = await axiosInstance.post<ApiResponse<User>>('/api/users', payload);
    return response.data;
  },

  // DELETE: Menghapus rekor berdasarkan parameter UID unik
  deleteUser: async (uid: string): Promise<ApiResponse<null>> => {
    const response = await axiosInstance.delete<ApiResponse<null>>(`/api/users/${uid}`);
    return response.data;
  }
};

02 Implementasi useMutation & Cache Invalidation (`src/pages/guru/useGuruLogic.ts`)

Perhatikan bagaimana fungsi queryClient.invalidateQueries digunakan di dalam callback onSuccess. Ini akan memaksa tabel data mendownload ulang list terbaru secara instan begitu proses simpan/hapus sukses. 

import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { useState } from 'react';
import { userApi } from '../../api/userApi';
import { User } from '../../types/user';
import axios from 'axios';

export function useGuruLogic() {
  const queryClient = useQueryClient();
  const [validationErrors, setValidationErrors] = useState<Record<string, string[]> | null>(null);

  // Read Operation
  const { data } = useQuery({
    queryKey: ['users', 'guru'],
    queryFn: () => userApi.getAllUsers('guru'),
  });

  // Write Operation (Create)
  const createUserMutation = useMutation({
    mutationFn: userApi.createUser,
    onSuccess: () => {
      // Invalidation: Otomatis memicu re-fetch data list guru di latar belakang
      queryClient.invalidateQueries({ queryKey: ['users', 'guru'] });
      setValidationErrors(null); // Bersihkan error lama jika sukses
    },
    onError: (error: unknown) => {
      // Menangkap error validasi form (HTTP 422) dari backend Laravel
      if (axios.isAxiosError(error) && error.response?.status === 422) {
        setValidationErrors(error.response.data.errors);
      }
    }
  });

  // Delete Operation
  const deleteUserMutation = useMutation({
    mutationFn: userApi.deleteUser,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['users', 'guru'] });
    }
  });

  return {
    guruList: data?.data || [],
    isSaving: createUserMutation.isPending,
    isDeleting: deleteUserMutation.isPending,
    validationErrors,
    addNewGuru: createUserMutation.mutate,
    deleteGuru: deleteUserMutation.mutate,
  };
}

03 Handling Error Form Pada Layer Tampilan (`src/pages/guru/GuruForm.tsx`)

Petakan objek validationErrors langsung di bawah input text field masing-masing untuk memberikan feedback visual merah jika input tidak memenuhi kriteria validasi backend. 

import React, { useState } from 'react';
import { useGuruLogic } from './useGuruLogic';
import { Button } from '@/components/ui/button';

export default function GuruForm(): React.JSX.Element {
  const { addNewGuru, isSaving, validationErrors } = useGuruLogic();
  
  const [formData, setFormData] = useState({
    nama_lengkap: '',
    email: '',
    role: 'guru' as const,
    nomor_identitas: ''
  });

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    addNewGuru(formData); // Kirim mutasi data ke server
  };

  return (
    <form onSubmit={handleSubmit} class="space-y-4 max-w-md bg-white p-6 rounded-xl border border-slate-100">
      <h3 class="font-bold text-base text-slate-900">Tambah Rekor Staff Baru</h3>
      
      {/* Input Nama */}
      <div class="flex flex-col gap-1">
        <label class="text-xs font-semibold text-slate-600">Nama Lengkap</label>
        <input 
          type="text" 
          value={formData.nama_lengkap}
          onChange={(e) => setFormData({...formData, nama_lengkap: e.target.value})}
          class={`p-2 border text-sm rounded-lg outline-none ${validationErrors?.nama_lengkap ? 'border-red-500' : 'border-slate-200'}`}
        />
        {/* Render pesan eror spesifik field nama_lengkap jika dikembalikan dari server */}
        {validationErrors?.nama_lengkap && (
          <span class="text-red-500 text-xs mt-0.5">{validationErrors.nama_lengkap[0]}</span>
        )}
      </div>

      {/* Input Nomor Identitas (NUPTK / Token Prefix) */}
      <div class="flex flex-col gap-1">
        <label class="text-xs font-semibold text-slate-600">Nomor Identitas</label>
        <input 
          type="text" 
          value={formData.nomor_identitas}
          onChange={(e) => setFormData({...formData, nomor_identitas: e.target.value})}
          class={`p-2 border text-sm rounded-lg outline-none ${validationErrors?.nomor_identitas ? 'border-red-500' : 'border-slate-200'}`}
        />
        {validationErrors?.nomor_identitas && (
          <span class="text-red-500 text-xs mt-0.5">{validationErrors.nomor_identitas[0]}</span>
        )}
      </div>

      <Button type="submit" disabled={isSaving} class="w-full">
        {isSaving ? 'Menyimpan ke Server...' : 'Simpan Rekor Data'}
      </Button>
    </form>
  );
}

MITIGASI PROSES ASYNC & MUTATION SECURITY

  • Mencegah Race Condition (Double Submit): Selalu kunci atau nonaktifkan (disabled={isPending}) tombol kirim form agar user tidak dapat melakukan penekanan tombol ganda saat request HTTP sedang berjalan.
  • Skema Data Kosong (Null-Safe Mapping): Saat menginisialisasi input, pastikan nilai awal tidak bernilai undefined melainkan string kosong ('') untuk menghindari peringatan un-controlled component dari React.
