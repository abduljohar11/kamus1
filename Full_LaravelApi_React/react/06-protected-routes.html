Protected Routes: Implementasi Guard Token Sanctum di React Router DOM

Amankan rute internal dashboard dari akses ilegal. Deteksi token otentikasi secara reaktif, kelola rute publik vs terproteksi, dan tendang otomatis pengguna tak dikenal langsung ke halaman login.

BAGAIMANA CARA KERJA CLIENT-SIDE ROUTE GUARD?

Sebelum sebuah halaman dashboard (seperti data guru atau laporan keuangan) dimuat oleh React Router, komponen peninjau akan memeriksa keberadaan token otentikasi (misalnya token Sanctum/Bearer):

  • Token Eksis: Komponen mengizinkan anak rute (<Outlet />) untuk di-render dan ditampilkan ke layar pengguna.
  • Token Kosong / Null: Komponen langsung mencegat eksekusi dan mengembalikan instruksi <Navigate to="/login" replace /> untuk membuang paksa sesi pengguna ke gerbang login luar.

01 Mendeteksi Status Token (`src/hooks/useAuth.ts`)

Buatlah sebuah fungsi pengawas terpusat untuk membaca token dari media penyimpanan lokal browser (*localStorage*) serta memeriksa status login pengguna secara cepat. 

import { useState } from 'react';

export function useAuth() {
  // Memeriksa token Sanctum dari localStorage secara real-time
  const [token, setToken] = useState<string | null>(() => {
    return localStorage.getItem('ajocell_token');
  });

  const isAuthenticated = !!token; // Konversi string menjadi boolean (true jika token ada)

  const logout = (): void => {
    localStorage.removeItem('ajocell_token');
    setToken(null);
  };

  return {
    token,
    isAuthenticated,
    logout
  };
}

02 Membuat Komponen Tameng Guard (`src/components/ProtectedRoute.tsx`)

Komponen ini bertugas memeriksa status otentikasi. Jika isAuthenticated bernilai salah (*false*), ia akan langsung memicu penendangan ke rute login menggunakan properti replace agar riwayat URL palsu tersebut tidak bisa diakses via tombol *back* browser. 

import React from 'react';
import { Navigate, Outlet } from 'react-router-dom';
import { useAuth } from '../hooks/useAuth';

export default function ProtectedRoute(): React.JSX.Element {
  const { isAuthenticated } = useAuth();

  // JIKA TIDAK ADA TOKEN: Langsung ditendang ke halaman login luar
  if (!isAuthenticated) {
    return <Navigate to="/login" replace />;
  }

  // JIKA ADA TOKEN: Izinkan lewat menuju sub-halaman dashboard di dalam layout
  return <Outlet />;
}

03 Penerapan Sarang Proteksi Pada Arsitektur URL (`src/router/AppRouter.tsx`)

Bungkus komponen Layout dan rute-rute penting internal dashboard di dalam sarang <Route element={<ProtectedRoute />}>. 

import React from 'react';
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom';
import Layout from '../components/Layout';
import ProtectedRoute from '../components/ProtectedRoute';
import Dashboard from '../pages/Dashboard';
import GuruList from '../pages/guru/GuruList';

export default function AppRouter(): React.JSX.Element {
  return (
    <BrowserRouter>
      <Routes>
        {/* Rute Terbuka (Public Route) */}
        <Route path="/login" element={<div>Halaman Login Form Sanctum</div>} />

        {/* BUNGKUSAN PROTEKSI UTAMA: Mengamankan Seluruh Halaman Dashboard */}
        <Route element={<ProtectedRoute />}>
          <Route path="/" element={<Layout />}>
            <Route index element={<Dashboard />} />
            <Route path="guru" element={<GuruList />} />
          </Route>
        </Route>

        {/* Tangkap sisa rute tidak dikenal */}
        <Route path="*" element={<Navigate to="/" replace />} />
      </Routes>
    </BrowserRouter>
  );
}

CATATAN KEAMANAN FRONTEND DEVELOPMENT

Perlu diingat bahwa mengamankan rute via JavaScript di sisi frontend *bukanlah* sistem keamanan absolut. Seseorang masih bisa memanipulasi kode lokal browser untuk memaksa masuk ke layout dashboard secara visual.

Keamanan Sejati: Terletak pada backend server (Middleware Laravel Sanctum). Jika ada pengguna ilegal berhasil menembus UI visual React tetapi tidak membawa Token Bearer asli di headers Axios-nya, API Laravel Anda harus tetap mengembalikan status HTTP 401 Unauthorized dan menolak pengiriman database.
