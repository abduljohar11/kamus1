Property & Method

Jika Class adalah sebuah mesin, maka Property adalah kabel dan bautnya (data), sedangkan Method adalah tombol-tombol yang bisa ditekan (fungsi).

1. Property: Atribut Objek

Property adalah variabel di dalam class. Untuk aplikasi guru, property bisa berupa nama siswa, NIS, atau nilai.

<?php
  class Siswa {
      public $nama; // Ini Property
      public $kelas;
  }

  $siswa1 = new Siswa();
  $siswa1->nama = "Andi"; // Mengisi Property
?>

2. Method: Perilaku Objek

Method adalah fungsi di dalam class. Gunakan $this untuk mengambil data dari property milik objek itu sendiri.

<?php
  class Siswa {
      public $nama = "Andi";

      public function sapa() {
          // Mengakses property menggunakan $this
          return "Halo, nama saya " . $this->nama;
      }
  }
?>

3. Hak Akses (Encapsulation)

Inilah bagian terpenting untuk keamanan. Kita bisa mengatur siapa saja yang boleh melihat atau mengubah data.

  • public : Bebas diakses dari mana saja (Luar & Dalam Class).
  • protected : Hanya bisa diakses oleh diri sendiri dan "Anak" (Class Turunan).
  • private : Rahasia! Hanya bisa diakses oleh diri sendiri di dalam class tersebut.

Studi Kasus: Keamanan Nilai

Bahaya: Jangan biarkan nilai bisa diubah langsung!

<?php
  class Raport {
      private $nilai = 0; // Private: Tidak bisa diubah sembarangan

      // Hanya guru yang bisa set nilai lewat tombol (Method) ini
      public function setNilai($angka) {
          if ($angka >= 0 && $angka <= 100) {
              $this->nilai = $angka;
          }
      }

      public function tampilkanNilai() {
          return "Nilai Anda: " . $this->nilai;
      }
  }

  $raportAndi = new Raport();
  // $raportAndi->nilai = 100; // ERROR! Karena private
  $raportAndi->setNilai(90);    // SUKSES! Lewat gerbang resmi
  echo $raportAndi->tampilkanNilai();
?>

Kesimpulan

Dengan menggunakan private atau protected, kamu melindungi integritas data aplikasi "Buku Guru" kamu. Pastikan data sensitif tidak bisa diakses langsung dari luar class!
