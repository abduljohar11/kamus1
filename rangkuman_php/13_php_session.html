PHP Session

Session adalah cara PHP "mengenali" siapa kamu saat berpindah dari satu halaman ke halaman lain. Tanpa Session, website akan "lupa" siapa kamu setiap kali kamu mengklik link baru.

Analogi Gelang Akses:
Bayangkan kamu masuk ke taman bermain. Di gerbang, kamu diberi gelang ID. Selama gelang itu ada di tanganmu, kamu bisa masuk ke wahana apa saja tanpa harus beli tiket lagi. Begitu gelang dilepas (Logout) atau taman tutup (Browser ditutup), aksesmu hilang.

1. Memulai Session

Agar PHP bisa membaca gelang ID tadi, kamu wajib menulis session_start() di baris paling atas kode kamu, bahkan sebelum tag HTML apapun.

<?php
  session_start(); // Wajib ada di baris pertama
?>

2. Menyimpan & Mengakses Data

Data disimpan dalam array superglobal $_SESSION. Sekali disimpan, data ini bisa dipanggil di file PHP mana pun selama session masih aktif.

<?php
  session_start();
  
  // Menyimpan data (Misal saat login berhasil)
  $_SESSION['username'] = "Pak Guru Ismail";
  $_SESSION['role'] = "admin";

  // Mengakses data di halaman lain
  echo "Selamat datang kembali, " . $_SESSION['username'];
?>

3. Menghapus Session (Logout)

Ada dua cara untuk menghapus data session:

  • unset(): Menghapus satu data spesifik.
  • session_destroy(): Menghapus seluruh data session (Cocok untuk tombol Logout).
<?php
  session_start();
  
  // Hapus satu data
  unset($_SESSION['temp_data']);

  // Logout total
  session_destroy();
  header("Location: login.php"); // Alihkan ke halaman login
?>

  }

  // Jika lolos, tampilkan konten rahasia
  echo "Halaman Rahasia Nilai Siswa";
?>

Kesimpulan

Session adalah kunci utama untuk membuat website yang memiliki sistem keamanan. Ingat: simpan data penting (seperti ID User) di Session (sisi Server), jangan di tempat yang bisa diubah oleh pengguna.
