Inheritance & Interface

1. Inheritance (Pewarisan)

Di Laravel, konsep ini adalah fondasi utama. Semua Controller yang kamu buat akan mewarisi fitur-fitur canggih dari Controller pusat Laravel.

Analogi: Ibarat sebuah **Template PowerPoint**. Kamu tidak membuat desain dari nol, kamu mengambil template yang sudah ada (Parent), lalu kamu tinggal menambah isi atau mengubah sedikit bagian (Child).
[Image of object-oriented programming inheritance diagram] 
<?php
  // Parent Class (Pusat Logika)
  class Controller {
      public function middleware() {
          return "Mengecek keamanan...";
      }
  }

  // Child Class (Spesifik untuk Siswa)
  class SiswaController extends Controller {
      public function index() {
          // Kita bisa panggil fungsi 'middleware' dari parent
          echo $this->middleware(); 
          return "Menampilkan daftar siswa.";
      }
  }

  $app = new SiswaController();
  echo $app->index();
?>

KENAPA INI PENTING DI LARAVEL?

Karena kamu akan sering menulis: class SiswaController extends Controller. Tanpa extends, controller kamu tidak punya kekuatan untuk melakukan validasi atau menampilkan view.

2. Interface (Kontrak Standardisasi)

Interface di Laravel sering disebut sebagai Contracts. Gunanya untuk memastikan bahwa class yang berbeda memiliki "cara pakai" yang sama.

Analogi: Ibarat **Colokan Listrik**. Tidak peduli merek kulkas atau TV kamu, colokannya harus sama (Interface). Jika tidak mengikuti standar colokan, perangkatmu tidak akan bisa mendapat listrik (Error). 
<?php
  // Perjanjian: Semua sistem Notifikasi HARUS punya fungsi 'kirim'
  interface NotificationInterface {
      public function kirim($pesan);
  }

  class EmailNotification implements NotificationInterface {
      public function kirim($pesan) {
          echo "Mengirim email: $pesan";
      }
  }

  class WhatsAppNotification implements NotificationInterface {
      public function kirim($pesan) {
          echo "Mengirim WA: $pesan";
      }
  }
?>

3. Ringkasan untuk Laravel

Inheritance (extends) Interface (implements)
Digunakan untuk **berbagi fungsi** yang sama. Digunakan untuk **mewajibkan** adanya fungsi tertentu.
Hubungan: "Adalah" (Guru *adalah* Pengguna). Hubungan: "Mampu" (Email *mampu* kirim notif).
Dipakai di: Controller & Model Laravel. Dipakai di: Service & Repository Pattern.

Kesimpulan

Pahami bahwa **Inheritance** adalah tentang siapa kamu (keturunan siapa), sedangkan **Interface** adalah tentang apa yang bisa kamu lakukan (mengikuti standar apa). Di Laravel, keduanya bekerja sama agar kodemu rapi dan tidak berantakan.
