PHP Security: Filter & Sanitasi

Jangan pernah percaya pada input pengguna! Orang bisa saja memasukkan data yang salah atau bahkan mencoba meretas website kamu melalui form. Kita butuh dua "Satpam" utama:

  • filter_var() Validation & Sanitization: Membersihkan karakter aneh atau mengecek kebenaran format data.
  • htmlspecialchars() Output Escaping: Memastikan teks yang ditampilkan di layar tidak dianggap sebagai kode program oleh browser.

1. Mengenal Macam-Macam Filter

PHP menyediakan banyak filter siap pakai untuk berbagai kebutuhan validasi data sekolah kamu:

Tipe Filter Nama Filter Kegunaan
VALIDATE FILTER_VALIDATE_EMAIL Cek format email (harus ada @ dan titik).
FILTER_VALIDATE_INT Cek apakah input beneran angka bulat (Integer).
FILTER_VALIDATE_URL Cek apakah input adalah alamat website yang sah.
SANITIZE FILTER_SANITIZE_EMAIL Buang karakter terlarang dari email.
FILTER_SANITIZE_NUMBER_INT Buang semua karakter kecuali angka, plus, dan minus.

2. Praktik Validasi Angka & Range

Dalam aplikasi guru, kita sering membatasi nilai siswa agar tidak kurang dari 0 atau lebih dari 100. Kita bisa menggunakan options pada filter_var.

<?php
  $nilai_input = $_POST['nilai'];

  // Pengaturan batas nilai
  $opsi = [
      "options" => [
          "min_range" => 0, 
          "max_range" => 100
      ]
  ];

  if (filter_var($nilai_input, FILTER_VALIDATE_INT, $opsi) !== false) {
      echo "Nilai $nilai_input sah dan disimpan.";
  } else {
      echo "Input salah! Nilai harus angka 0 - 100.";
  }
?>

3. Penetral XSS dengan htmlspecialchars()

XSS adalah serangan di mana hacker memasukkan kode JavaScript ke form. htmlspecialchars() mengubah karakter seperti < menjadi &lt; sehingga browser hanya menampilkannya sebagai teks biasa.

Penting: Gunakan ini SETIAP KALI kamu melakukan "echo" data yang berasal dari input pengguna! 
<?php
  $nama_kotor = "<script>alert('Hack!')</script>";
  
  // Membersihkan agar aman ditampilkan di layar
  $nama_aman = htmlspecialchars($nama_kotor, ENT_QUOTES, 'UTF-8');

  echo "Nama: " . $nama_aman; 
  // Browser akan mencetak teks <script>... di layar, bukan menjalankan alert.
?>

Ikuti urutan ini agar aplikasi "Buku Guru" kamu aman maksimal:

  1. Ambil data dari $_POST.
  2. Sanitasi (bersihkan) jika perlu menggunakan FILTER_SANITIZE.
  3. Validasi (cek format) menggunakan FILTER_VALIDATE.
  4. Simpan ke database (Gunakan Prepared Statements - akan dipelajari nanti).
  5. Saat ingin menampilkan ke layar, gunakan htmlspecialchars().

Kesimpulan

Keamanan bukan sebuah fitur, tapi sebuah keharusan. Dengan tabel filter di atas, kamu bisa memastikan hanya data yang "sehat" yang masuk ke sistem aplikasi kamu.
