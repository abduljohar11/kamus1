PHP $_GET & $_POST

Agar aplikasi "Buku Guru" bisa menerima input seperti nama siswa atau nilai ujian, kita butuh cara untuk mengirim data dari browser ke server. Di sinilah $_GET dan $_POST berperan.

Analogi Sederhana:

GET itu seperti Kartu Pos: Isinya terlihat oleh siapa saja (di URL) karena terbatas dan terbuka.

POST itu seperti Surat dalam Amplop: Isinya tertutup rapat di dalam (Request Body), tidak terlihat dari luar.

1. Method $_GET (Lewat URL)

Data dikirim dengan cara ditempelkan di belakang alamat URL. Cocok untuk fitur pencarian atau link navigasi.

<?php
  // URL: proses.php?nama=Budi&kelas=10A
  
  if (isset($_GET['nama'])) {
      echo "Mencari data siswa: " . $_GET['nama'];
  }
?>

2. Method $_POST (Lewat Body)

Data dikirim secara "tersembunyi". Wajib digunakan untuk data sensitif atau data yang panjang seperti form input nilai atau login.

<?php
  // Data tidak terlihat di URL
  if ($_SERVER["REQUEST_METHOD"] == "POST") {
      $nilai = $_POST['nilai_siswa'];
      echo "Nilai berhasil disimpan: " . $nilai;
  }
?>

Perbandingan GET vs POST

Fitur $_GET $_POST
Keamanan Rendah (Terlihat di URL) Lebih Tinggi (Tersembunyi)
Kapasitas Terbatas (maks 2000 karakter) Tidak Terbatas
Kegunaan Pencarian, Filter, Link Simpan Data, Login, Upload

Beginilah cara kamu menggabungkan Form HTML dengan PHP nantinya:

<!-- Form di file index.php -->
<form action="proses.php" method="POST">
    Nama: <input type="text" name="nama_siswa">
    Nilai: <input type="number" name="skor">
    <button type="submit">Simpan Nilai</button>
</form>

<?php
  // Di file proses.php
  if (isset($_POST['nama_siswa'])) {
      $nama = $_POST['nama_siswa'];
      $skor = $_POST['skor'];
      echo "Siswa $nama mendapatkan nilai $skor";
  }
?>

Validasi isset()

Selalu gunakan isset() sebelum memanggil data agar tidak muncul error "Undefined Index" jika user langsung membuka halaman tanpa mengisi form.

Kesimpulan

Gunakan POST untuk formulir yang mengubah data di database (seperti input nilai), dan gunakan GET untuk link yang hanya menampilkan data (seperti melihat detail siswa).
