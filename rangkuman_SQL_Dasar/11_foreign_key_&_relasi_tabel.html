FOREIGN KEY & RELASI

Hubungan Antar Tabel

FOREIGN KEY adalah "tali pengikat" yang menghubungkan data di satu tabel dengan data di tabel lain secara resmi dan aman.

Analogi Sertifikat Siswa:
Bayangkan tabel JURUSAN dan tabel SISWA:
  • Tabel JURUSAN punya ID 1 (IPA) dan ID 2 (IPS).
  • Tabel SISWA tidak menulis "IPA" berkali-kali, tapi cukup menulis "ID 1" di kolom jurusan_id.
  • Foreign Key: Memastikan tidak ada siswa yang menulis ID 3, karena ID 3 tidak terdaftar di tabel JURUSAN.

Penerapan Nyata

-- Tabel Utama (Parent)
CREATE TABLE jurusan (
  id INT PRIMARY KEY AUTO_INCREMENT,
  nama_jurusan VARCHAR(50)
);

-- Tabel Terikat (Child)
CREATE TABLE siswa (
  id INT PRIMARY KEY AUTO_INCREMENT,
  nama VARCHAR(100),
  jurusan_id INT, -- Kolom penghubung
  FOREIGN KEY (jurusan_id) REFERENCES jurusan(id)
);
