Responsive Design

Tailwind menggunakan pendekatan Mobile-First. Artinya, class tanpa prefix akan berlaku di layar terkecil, dan prefix seperti md: akan menimpanya saat layar melebar.

1. Daftar Breakpoint

Tailwind menyediakan 5 breakpoint standar untuk menargetkan perangkat yang berbeda:

PrefixMinimum WidthTarget Perangkat
sm:640pxSmartphone (Landscape)
md:768pxTablet
lg:1024pxLaptop / Desktop Kecil
xl:1280pxDesktop Besar
2xl:1536pxMonitor Ultra Wide

2. Konsep Mobile First

Alih-alih menulis media query untuk layar kecil, di Tailwind kita menulis class dasar untuk HP, lalu menambah prefix untuk layar yang lebih lebar.

<!-- text-sm (HP), md:text-xl (Tablet), lg:text-3xl (Laptop) -->
<h1 class="text-sm md:text-xl lg:text-3xl font-bold">
  Judul Responsif
</h1>

3. Simulasi Grid Responsif

Gunakan prefix pada grid untuk mengubah jumlah kolom berdasarkan ukuran layar. Contoh: 1 kolom di HP, 2 kolom di Tablet, dan 3 kolom di Laptop.

Item 1
Item 2
Item 3
<!-- 1 Kolom di HP, 2 di Tablet, 3 di Laptop -->
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <div>1</div>
  <div>2</div>
  <div>3</div>
</div>

4. Visibility (Show/Hide)

Kamu bisa menyembunyikan elemen di HP dan memunculkannya di Desktop menggunakan class hidden dan block.

<!-- Tombol ini hanya muncul di layar Tablet ke atas -->
<button class="hidden md:block bg-blue-600 text-white p-2">
  Menu Desktop
</button>

Kesimpulan

Sistem responsif Tailwind sangat efisien karena kita tidak perlu lagi pindah ke file CSS untuk menulis @media (min-width: ...). Cukup gunakan prefix dan desainmu akan langsung menyesuaikan perangkat pengguna.
