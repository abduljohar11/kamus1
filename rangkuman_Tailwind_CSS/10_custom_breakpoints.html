Custom Breakpoints (v4)

Tailwind CSS v4 memperkenalkan pendekatan CSS-first. Semua konfigurasi variabel, termasuk breakpoint, kini ditulis langsung di dalam file CSS utama menggunakan direktif @theme.

1. Breakpoint Bawaan v4

Berikut adalah standar breakpoint yang otomatis aktif saat kamu menginstal Tailwind v4:

Variabel CSSPrefixWidth
--breakpoint-smsm:640px
--breakpoint-mdmd:768px
--breakpoint-lglg:1024px
--breakpoint-xlxl:1280px
--breakpoint-2xl2xl:1536px

2. Menambahkan Breakpoint Baru

Untuk menambahkan ukuran baru (misalnya xs untuk HP kecil atau 3xl untuk layar raksasa), tuliskan variabel di dalam blok @theme di file CSS kamu:

@import "tailwindcss";

@theme {
  --breakpoint-xs: 480px;
  --breakpoint-3xl: 1920px;
}

Setelah ditambahkan, kamu bisa langsung menggunakannya di HTML:

<div class="grid grid-cols-1 xs:grid-cols-2 3xl:grid-cols-6">
  <!-- Konten adaptif -->
</div>

3. Arbitrary Breakpoints

Jika kamu hanya butuh satu ukuran khusus tanpa mau repot mengatur variabel CSS, gunakan Arbitrary Values dengan tanda kurung siku:

<!-- Aktif hanya saat layar minimal 450px -->
<div class="min-[450px]:bg-blue-500">
  Background berubah di 450px
</div>

4. Menghapus Breakpoint Default

Jika kamu ingin membuang semua breakpoint bawaan Tailwind dan membuat sistem sendiri, gunakan kata kunci null:

@theme {
  --breakpoint-*: initial; /* Menghapus semua default */
  --breakpoint-tablet: 768px;
  --breakpoint-desktop: 1024px;
}

Kesimpulan

Tailwind v4 membuat konfigurasi jauh lebih cepat karena kita tetap berada di dalam ekosistem CSS. Ingat, fitur @theme ini membutuhkan Tailwind CLI atau Vite Plugin agar bisa diproses.
