Variabel & Extend Theme

Tailwind CSS v4 memperkenalkan model CSS-first. Kini, konfigurasi tema tidak lagi wajib di file JavaScript, melainkan bisa langsung didefinisikan di dalam CSS menggunakan direktif @theme.

1. Apa itu Variabel Tema?

Dengan menuliskan variabel CSS di dalam blok @theme, Tailwind akan otomatis mendeteksi variabel tersebut dan menciptakan utility class baru untukmu.

@theme {
  --color-mint: #3ff2d7;
  --color-brand-primary: #2563eb;
}

Secara otomatis, kamu kini memiliki akses ke class: bg-mint, text-mint, border-brand-primary, dsb.

2. Pemetaan Token ke Utility

Tailwind menggunakan prefix variabel tertentu untuk menentukan jenis utility yang dihasilkan:

Prefix VariabelUtility yang DihasilkanContoh Penggunaan
--color-*bg-*, text-*, border-*bg-mint
--spacing-*p-*, m-*, w-*, h-*p-72
--font-*font-*font-display
--breakpoint-*sm:, md:, lg:, ...3xl:hidden
--radius-*rounded-*rounded-huge

3. Extend via CSS (@theme)

Untuk menambah atau menimpa tema bawaan, kamu cukup menuliskannya di file CSS utama. Ini jauh lebih ringan daripada mengedit tailwind.config.js.

@theme {
  /* Menambah ukuran spacing baru */
  --spacing-72: 18rem;
  
  /* Menambah font kustom */
  --font-display: "Oswald", sans-serif;
  
  /* Menambah breakpoint kustom */
  --breakpoint-3xl: 1920px;
}

4. Dynamic & Arbitrary Values

V4 tetap mendukung nilai dinamis yang bisa dipadukan dengan variabel CSS yang sudah kamu definisikan sebelumnya.

<!-- Menggunakan variabel tema kustom -->
<div class="p-72 bg-mint text-brand-primary font-display">
  Konten dengan desain token kustom.
</div>

<!-- Menggunakan nilai sembarang (Arbitrary) -->
<div class="bg-[#bada55] w-[500px]">
  Nilai sekali pakai.
</div>

Kesimpulan

Pendekatan CSS-first melalui @theme membuat workflow menjadi lebih konsisten. Kamu bisa memusatkan semua aturan desain di dalam file CSS, menjadikannya lebih mudah dibaca dan dikelola tanpa harus berpindah ke file konfigurasi JavaScript.
