Tips & Best Practices

Tailwind CSS sangat kuat, namun tanpa disiplin, HTML kita bisa menjadi "berantakan". Berikut adalah panduan agar proyekmu tetap skalabel dan profesional.

1. Konsistensi Urutan Class

Pernahkah kamu bingung mencari class warna di antara puluhan class lainnya? Gunakan urutan standar berikut:

KategoriContoh Class
Layout & Posisiflex, relative, top-0, z-10
Box Modelm-4, p-6, w-full, h-auto
Tipografitext-lg, font-bold, leading-relaxed
Visualbg-blue-500, border, rounded-xl, shadow
Interaktifhover:bg-blue-600, transition

2. Ekstraksi Komponen (@apply)

Jika kamu memiliki elemen yang dipakai berulang kali (misal: tombol), jangan duplikasi class di HTML. Gunakan direktif @apply di file CSS proyekmu.

/* style.css */
.btn-primary {
  @apply bg-blue-600 text-white px-4 py-2 rounded-lg hover:bg-blue-700 transition;
}

Lalu gunakan class pendek di HTML:

<button class="btn-primary">Klik Saya</button>

3. Manfaatkan Plugin Resmi

Tailwind memiliki plugin yang didesain khusus untuk menangani elemen HTML yang "sulit" di-style secara default.

  • @tailwindcss/forms: Me-reset elemen input agar mudah di-style.
  • @tailwindcss/typography: Memberikan class prose untuk artikel.
  • @tailwindcss/aspect-ratio: Mengunci rasio gambar/video.

4. Gunakan Prettier Plugin

Sangat disarankan menginstal plugin Prettier for Tailwind CSS di VS Code. Plugin ini otomatis akan mengurutkan class-mu agar rapi setiap kali kamu menyimpan file (save).

5. Dark Mode & Token Desain

Selalu gunakan token desain di @theme (Tailwind v4) untuk menjaga konsistensi warna. Hindari menggunakan warna sembarang (arbitrary hex) terlalu sering agar tema mudah diganti di masa depan.

Kesimpulan

Tailwind bukan hanya tentang menulis class, tapi tentang sistem desain. Dengan menjaga urutan class, mengekstraksi komponen yang berulang, dan menggunakan plugin resmi, proyekmu akan jauh lebih mudah dikelola.
