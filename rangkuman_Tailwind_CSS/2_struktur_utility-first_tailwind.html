Struktur Utility-First Tailwind

Dengan pendekatan utility-first, kita tidak lagi menulis aturan CSS secara manual di file terpisah. Sebagai gantinya, kita menggunakan banyak utility class langsung di elemen HTML.

Apa Itu Utility Class?

Setiap class di Tailwind hanya memiliki satu fungsi spesifik (seperti margin saja, atau warna saja). Kita menggabungkannya seperti menyusun blok bangunan (LEGO) untuk membuat UI.

<!-- Contoh penggunaan Utility Class -->
<button class="bg-blue-500 text-white p-4 rounded-lg shadow-md">
  Klik Saya
</button>

Struktur Penulisan

Walaupun atribut class terlihat panjang, ini memberikan kontrol penuh tanpa perlu berpindah-pindah file.

<div class="flex items-center justify-between p-6 bg-white rounded-xl shadow-lg border border-gray-200">
  <p class="text-gray-700 font-medium">Konten Terpusat</p>
</div>

Tools untuk Belajar

Gunakan tools interaktif ini untuk mencoba Tailwind secara langsung di browser:

  • play.tailwindcss.com (Resmi)
  • playground.santrikoding.com (Lokal)

Contoh Lengkap: Kartu Informasi

Berikut adalah implementasi card sederhana menggunakan Tailwind CSS:

<div class="max-w-sm mx-auto bg-white rounded-2xl shadow-xl p-8">
  <h3 class="text-xl font-bold text-blue-600 mb-2">Tailwind Card</h3>
  <p class="text-gray-600">Dibuat tanpa menulis satu barispun kode CSS manual.</p>
</div>

Kesimpulan

Pendekatan utility-first membuat workflow pengembangan UI menjadi jauh lebih cepat dan konsisten. Kita cukup fokus pada struktur HTML kita.
