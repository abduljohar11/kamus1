Variable (var, let, const)

Apa Itu Variabel?

Variabel adalah sebuah wadah atau tempat penyimpanan data di memori komputer. Bayangkan variabel sebagai sebuah "kotak" yang memiliki nama, dan di dalam kotak tersebut Anda bisa menyimpan informasi untuk digunakan berkali-kali.

Bayangkan sebuah Botol Minum. Label pada botol (misal: "Minuman") adalah Nama Variabel, dan air di dalamnya adalah Nilai/Data. Anda bisa mengganti air putih dengan jus (Re-assign), atau Anda bisa menggunakan botol yang tidak bisa dibuka tutupnya (Const).

Cara Mendeklarasikan Variabel

JavaScript menyediakan tiga cara untuk membuat variabel. Dulu hanya ada var, namun sejak standar ES6 (2015), muncul let dan const yang jauh lebih aman.

1. var

Ini adalah cara lama. var memiliki perilaku yang unik (dan sering membingungkan) yang disebut hoisting.

  • Function-scoped: Hanya terkurung di dalam fungsi.
  • Bisa di-redeclare: Anda bisa membuat variabel dengan nama yang sama berkali-kali tanpa error.
var nama = "Budi";
var nama = "Siti"; // Tidak error, Budi terhapus
console.log(nama); // Output: Siti

2. let

Solusi modern untuk variabel yang nilainya akan berubah (seperti skor game atau counter).

  • Block-scoped: Terkurung di dalam kurung kurawal { }.
  • Tidak bisa di-redeclare: Mencegah bug karena nama variabel yang sama.
let skor = 10;
skor = 20; // Boleh (Re-assign)

// let skor = 30; // ERROR! Tidak boleh dideklarasikan ulang

3. const

Digunakan untuk nilai yang tetap dan tidak akan pernah diubah (seperti Nama Brand, API Key, atau rumus matematika).

  • Block-scoped: Sama seperti let.
  • Immutable: Nilainya tidak bisa diganti (Re-assign).
  • Wajib diisi: Harus langsung diberi nilai saat dibuat.
const API_KEY = "12345ABC";
// API_KEY = "67890DEF"; // ERROR!

Kapan Harus Pakai yang Mana?

Aturan emas developer modern:

  • Gunakan const secara default untuk semua variabel.
  • Jika variabel tersebut memang perlu diubah nilainya nanti, ganti menjadi let.
  • Lupakan var, kecuali Anda sedang memperbaiki website dari tahun 2010.

Kesimpulan

Memilih antara let dan const adalah cara termudah untuk membuat kode Anda lebih stabil dan mudah dibaca oleh orang lain. Dengan menggunakan const, Anda memberi tahu developer lain bahwa "Nilai ini penting dan tidak boleh berubah".
