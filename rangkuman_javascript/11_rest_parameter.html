Rest Parameter

Apa Itu Rest Parameter?

Rest Parameter digunakan untuk mengumpulkan sisa argumen ke dalam sebuah array tunggal. Meskipun simbolnya sama dengan Spread Operator (...), fungsinya adalah kebalikannya.

ðŸ’¡ Analogi

Bayangkan sebuah Net/Saringan di kolam bola. Bola-bola yang dilemparkan (argumen fungsi) akan ditangkap dan dikumpulkan ke dalam satu wadah (array). Spread menyebarkan bola keluar dari kotak, sedangkan Rest menangkap semua bola yang tersisa dan memasukkannya ke dalam kotak.

1. Menangkap Banyak Argumen

Biasanya, fungsi hanya menerima jumlah parameter yang pasti. Dengan Rest Parameter, kita bisa menerima jumlah input yang tak terbatas.

// ...angka akan mengumpulkan semua input menjadi array [1, 2, 3, 4]
function jumlahkanSemua(...angka) {
  return angka.reduce((total, n) => total + n, 0);
}

console.log(jumlahkanSemua(1, 2, 3, 4)); // 10

2. Rest Parameter Harus di Akhir

Aturannya ketat: Rest parameter wajib diletakkan di posisi paling terakhir dalam daftar parameter fungsi. Anda tidak boleh meletakkan parameter lain setelahnya.

function sapaJuara(juara1, juara2, ...pesertaLain) {
  console.log(`Emas: ${juara1}`);
  console.log(`Perak: ${juara2}`);
  console.log(`Peserta lainnya: ${pesertaLain.join(", ")}`);
}

sapaJuara("Andi", "Budi", "Cici", "Dedi", "Eka");

3. Rest pada Destructuring

Sangat berguna saat Anda ingin mengambil beberapa nilai dari data dan menyimpan sisanya ke variabel lain.

Array Destructuring

const angka = [1, 2, 3, 4, 5];
const [pertama, kedua, ...sisanya] = angka;

console.log(sisanya); // [3, 4, 5]

Object Destructuring

const user = { nama: "Siti", umur: 20, kota: "Bandung" };
const { nama, ...infoLain } = user;

console.log(infoLain); // { umur: 20, kota: 'Bandung' }

Perbedaan Utama

Fitur Spread Operator Rest Parameter
Tujuan Menyebar isi ke luar Mengumpulkan sisa ke dalam
Lokasi Pakai Saat memanggil fungsi / di dalam array/obj Saat definisi fungsi / destructuring
Hasil Akhir Nilai individual Sebuah Array atau Object baru

Kesimpulan

Rest Parameter (...) adalah cara modern untuk menangani data yang jumlahnya tidak menentu secara dinamis. Ia membuat kode Anda lebih fleksibel dan bersih, terutama saat bekerja dengan fungsi-fungsi pengolahan data.

Di artikel berikutnya, kita akan membahas Destructuring & Restructuring: Teknik tingkat lanjut untuk memanipulasi data dengan sangat efisien!
