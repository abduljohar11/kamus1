Destructuring & Restructuring

Apa Itu Destructuring?

Destructuring adalah fitur ES6 yang memungkinkan kita untuk membongkar nilai dari array atau object ke dalam variabel individu dengan sintaksis yang sangat singkat.

ðŸ’¡ Analogi

Bayangkan Anda memiliki sebuah Koper (Array/Object) yang berisi baju, celana, dan sepatu. Tanpa Destructuring, Anda harus membuka koper, mencari satu per satu, dan meletakkannya di lemari. Dengan Destructuring, Anda seolah-olah memiliki kekuatan sihir untuk langsung memindahkan semua isi koper ke tempatnya masing-masing hanya dengan satu jentikan jari.

1. Destructuring Array

Alih-alih mengakses index satu per satu menggunakan [0], kita langsung mendefinisikan nama variabel di dalam kurung siku.

const warna = ["Merah", "Hijau", "Biru"];

// Destructuring
const [w1, w2, w3] = warna;

console.log(w1); // Merah
console.log(w2); // Hijau

Skip Item & Rest Parameter

const angka = [1, 2, 3, 4, 5];

// Melewati angka 2, dan mengambil sisanya
const [satu, , tiga, ...sisanya] = angka;

console.log(satu);    // 1
console.log(tiga);    // 3
console.log(sisanya); // [4, 5]

2. Destructuring Object

Pada object, kita menggunakan kurung kurawal {}. Nama variabel harus sesuai dengan nama key properti di dalam object tersebut.

const user = { nama: "Rudi", umur: 30 };

// Destructuring
const { nama, umur } = user;

console.log(nama); // Rudi

Alias (Mengganti Nama Variabel)

const { nama: namaLengkap } = user;
console.log(namaLengkap); // Rudi

Apa Itu Restructuring?

Restructuring adalah proses mengumpulkan kembali variabel-variabel individu menjadi sebuah Object atau Array. Di JavaScript modern, jika nama variabel sama dengan nama key object, kita cukup menulisnya sekali.

const merk = "Toyota";
const tahun = 2024;

// Restructuring (Shorthand Property)
const mobil = { merk, tahun };

console.log(mobil); // { merk: "Toyota", tahun: 2024 }

Kesimpulan

Destructuring dan Restructuring adalah teknik wajib bagi developer JavaScript modern. Fitur ini sangat sering digunakan dalam pengerjaan React, terutama saat mengelola Props dan State agar kode tetap bersih dan mudah dibaca.
