Seeding

  • Membuat Seeder
  • Menjalankan Seeder
  • Menjalankan Beberapa Seeder

Seeding di dalam Laravel merupakan fitur yang digunakan untuk membuat data awal di dalam database. Biasanya digunakan untuk melakukan testing terhadap suatu fitur di dalam aplikasi menggunakan live data. Dan semua class seeding yang dibuat akan diletakkan di dalam folder database/seeders.

Membuat Seeder

Untuk membuat sebuah seeder, kita bisa menggunakan perintah artisan make:seed dan diikuti nama seeder yang akan dibuat. Contohnya seperti berikut ini.

php artisan make:seeder UserSeeder

Jika perintah seeder di atas berhasil dijalankan, maka kita akan mendapatkan file baru dengan nama UserSeeder yang berada di dalam folder database/seeders.

Jika kita buka file tersebut, maka hasilnya seperti berikut ini.

<?php

namespace Database\Seeders;

use Illuminate\Database\Console\Seeds\WithoutModelEvents;
use Illuminate\Database\Seeder;

class UserSeeder extends Seeder
{
    /**
     * Run the database seeds.
     *
     * @return void
     */
    public function run(): void
    {
        //
    }
}

Ketika membuat class seeder, maka akan terdapat 1 method bawaan yang bernama function run. Di dalam method tersebut kita bisa gunakan untuk melakukan proses insert data ke dalam database.

Contohnya seperti berikut ini.

public function run()
{
    \DB::table('users')->insert([
        'name'      => 'Fika Ridaul Maulayya,
        'email'     => 'santrikoding@gmail.com',
        'password'  => bcrypt('password'),
    ]);
}

Menjalankan Seeder

Untuk menjalankan proses seeding, kita bisa menggunakan perintah berikut ini.

php artisan db:seed --class=UserSeeder

Menjalankan Beberapa Seeder

Jika ada pertanyaan, bagaimana jika file seeder-nya banyak ? apakah kita juga harus menjalankan perintah seeding-nya satu-satu ?.

Tentu tidak, kita bisa menjalankan beberapa seeder dalam waktu yang bersamaan. Jika kita perhatikan, kita sudah disediakan file yang bernama DatabaseSeeder.php yang berada di dalam folder database/seeders. File tersebut yang nanti digunakan untuk menjalankan banyak file seeder sekaligus.

Contohnya seperti berikut ini.

<?php

namespace Database\Seeders;

// use Illuminate\Database\Console\Seeds\WithoutModelEvents;
use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    /**
     * Seed the application's database.
     *
     * @return void
     */
    public function run(): void
    {
        $this->call([
            UserSeeder::class,

            //seeder lainnya
        ]);
    }
}

Di atas, pada method function run kita memanggil method call dan isinya adalah class UserSeeder yang telah kita buat.

Jika banyak file seeder, tinggal ditambahkan saja dibawahnya, kurang lebih seperti berikut ini contohnya.

$this->call([
    UserSeeder::class,
    CategorySeeder::class,
    ProductSeeder::class,
]);

Jika sudah seperti itu, maka untuk menjalankan proses seeding hanya cukup seperti berikut ini.

php artisan db:seed
