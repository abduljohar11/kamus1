Middleware

  • Membuat Middleware
  • Mendaftarkan Middleware
  • Parameter di Middleware

Middleware merupakan mekanisme yang digunakan untuk melakukan filtering terhadap permintaan HTTP ke dalam aplikasi. Contoh sederhananya digunakan untuk melakukan pengecekan terhadap user apakah sudah melakukan proses otentikasi. Jika belum maka middleware akan mengarahkan pada halaman login. Namun jika user sudah melakukan otentikasi, maka middleware akan mengizinkan permintaan ke dalam aplikasi.

Middleware tidak hanya digunakan untuk pengecekan otentikasi, namun bisa digunakan untuk semua jenis kondisi, misalnya menentukan halaman tertentu hanya bisa diakses jika user tersebut memiliki usia di atas 20 tahun, dan lain-lain. Semua Middleware di Laravel disimpan di dalam folder app/Http/Middleware.

Membuat Middleware

Untuk membuat middleware di dalam Laravel, kita bisa menggunakan perintah make:middleware. Contohnya seperti berikut ini.

php artisan make:middleware CheckAge

Perintah di atas digunakan untuk membuat middleware baru dengan nama CheckAge dan jika perintah di atass berhasil dijalankan, maka akan membuat file baru dengan nama CheckAge.php di dalam folder app/Http/Middleware.

Kurang lebih seperti berikut ini hasilnya.

<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class CheckAge
{
    /**
     * Handle an incoming request.
     *
     * @param  \Illuminate\Http\Request  $request
     * @param  \Closure(\Illuminate\Http\Request): (\Illuminate\Http\Response|\Illuminate\Http\RedirectResponse)  $next
     * @return \Illuminate\Http\Response|\Illuminate\Http\RedirectResponse
     */
    public function handle(Request $request, Closure $next)
    {
        return $next($request);
    }
}

Bisa kita lihat, di dalam middleware di atas kita akan diberikan method yang bernama handle. Method tersebut yang nanti kita gunakan untuk menulis logika aplikasi. Di dalam method handle tersebut memiliki 2 parameter, yaitu Request $request dan Closure $next.

Berikut ini contoh logika untuk mengizinkan user jika sudah memiliki umur di atas 20 tahun.

public function handle(Request $request, Closure $next)
{
    if($request->age < 20 ) {
        return redirect()->route('home')->with('errors', 'Umur kamu masih dibawah 20 tahun!');
    }

    return $next($request);
}

Di atas, kita membuat kondisi apabila umur user dibawah 20 tahun, maka kita akan arahkan ke dalam route yang bernama home dan memberikan pesan Umur kamu masih dibawah 20 tahun!.

Namun jika user memiliki umur di atas 20 tahun, maka akan dilanjutkan ke dalam controller yang dituju menggunakan $next($request).

Mendaftarkan Middleware

Jika kita ingin menjadikan middleware bersifat global, yang artinya akan selalu dieksekusi saat ada permintaan Http ke aplikasi, maka kita perlu mendaftarkan middleware tersebut di dalam file bootstrap/app.php.

Kemudian jika kita ingin agar bisa menggunakan middleware di dalam route, maka kita perlu mendaftarkan di dalam file bootstrap/app.php.

Contohnya seperti berikut ini.

use App\Http\Middleware\CheckAge;
 
->withMiddleware(function (Middleware $middleware) {
    $middleware->alias([
        'checkAge' => CheckAge::class
    ]);
})

Di atas, kita membuat key checkAge dan kita arahkan ke dalam class middleware itu berada.

Setelah mendaftarkan middleware seperti di atas, maka kita sudah bisa menggunakan di dalam route. Contohnya seperti berikut ini.

Route::get('/profile', function () {
    //
})->middleware('checkAge');

Dari penulisan route di atas, artinya user yang bisa mengakses /profile adalah user yang memiliki usia di atas 20 tahun. Karena di dalam route di atas kita menambahkan middleware checkAge.

Parameter di Middleware

Middleware juga bisa menerima parameter tambahan. Misalnya kita gunakan untuk memeriksa role dari user yang telah melakukan otentikasi.

Contohnya seperti berikut ini.

public function handle($request, Closure $next, $role)
{
    if (! $request->user()->hasRole($role)) {
    	// Redirect...
    }

    return $next($request);
}

Di atas, kita menambahkan parameter $role di dalam method handle. Dan di dalam method handle kita membuat kondisi untuk memerika role dari user tersebut berdasarkan nilai dari parameter.

Dan untuk menggunakan di dalam route, kita bisa seperti berikut ini.

Route::get('/dashboard', function () {
    //
})->middleware('role:admin');



Route::get('/post/{id}', function ($id) {
    //
})->middleware('role:author');

Di atas, kita bisa memberikan nilai middleware secara dinamis menggunakan :, seperti contoh berikut ini.

role:admin

role:author
