Di dalam Laravel ada fitur yang bernama route. Route ini digunakan sebagai penghubung antara user dengan aplikasi. Maksudnya, URL yang kita tulis di dalam browser akan melewati yang namanya route. Dan di dalam route tersebut baru akan ditentukan mau diarahkan kemana, apakah ke controller atau langsung ke dalam view.

Konfigurasi route di dalam Laravel ada beberapa jenis, tapi yang akan sering kita gunakan yaitu web dan api. Keduanya berada di dalam folder berikut ini.

  1. routes/web.php - digunakan untuk route web.
  2. routes/api.php - digunakan untuk route Rest API.

Berikut ini contoh sederhana penulisan route di Laravel. 

use Illuminate\Support\Facades\Route;
 
Route::get('/santri', function () {
    return 'Santri Koding';
});

Di atas, kita melakukan definisi route baru dengan path / URL /santri. Dan route tersebut menggunakan method GET.

Jika route di atas diakses di dalam browser, maka akan menampilkan kata Santri Koding, karena di dalam route tersebut kita melakukan return ke dalam text Santri Koding.

Jenis-jenis Method Route

Route memiliki beberapa jenis method, berikut ini penjelasannya.

METHOD DESCRIPTION RESPONSE
GET route yang bisa diakses langsung melalui URL browser dan digunakan untuk menampilkan list data dari database. 200 - Ok
POST route yang biasanya digunakan di dalam form untuk proses insert data ke dalam database 201 - Created
PUT route yang biasa digunakan di dalam form untuk proses update data ke database. 200 - Ok
PATCH route yang biasa digunakan di dalam form untuk proses update data ke database. 200 - Ok
DELETE route yang biasa digunakan untuk hapus data dari database. 200 - Ok 
Route::get($uri, $callback);
Route::post($uri, $callback);
Route::put($uri, $callback);
Route::patch($uri, $callback);
Route::delete($uri, $callback);

Route Parameter

Di Laravel ada 2 jenis parameter route, yaitu parameter yang wajib diisi dan parameter yang tidak wajib diisi (opsional).

Parameter biasanya kita gunakan untuk mengambil value dari suatu data, misalnya kita ingin mengambil nilai ID dari user di URL browser.

Parameter yang Wajib Diisi

Berikut ini contoh penulisan route Laravel dengan parameter yang wajib diisi. 

Route::get('/user/{id}', function ($id) {
    return 'User '.$id;
});

Untuk contoh route di atas, kita bisa menjalankan di browser dengan URL http://localhost:8000/user/{id}. Nilai {id} bisa kita tulis berapapun, misalnya angka 1.

Maka hasilnya akan seperti berikut ini.

Parameter yang Tidak Wajib Diisi

Kadang kita juga membutuhkan parameter yang sifatnya opsional, yaitu bisa diisi atau juga bisa dikosongkan nilai-nya.

Di dalam Laravel, untuk membuat parameter yang bersifat opsional kita hanya perlu menambahkan karakter ? di dalam definisi route-nya.

Berikut ini contoh penulisan parameter yang bersifat opsional. 

Route::get('/user/{name?}', function ($name = null) {
    return $name;
});

Dari penulisan route di atas, pada parameter name kita tambahkan karakter ? yang menandakan parameter tersebut bersifat opsional.

Dan di dalam clousure function kita definisikan default value jika parameter tersebut tidak diisi, yaitu null. Kita juga bisa menambahkan value default lainnya, misalnya seperti berikut ini. 

Route::get('/user/{name?}', function ($name = 'John') {
    return $name;
});

Jika dijalankan di dalam browser tanpa menambahkan parameter, maka hasilnya akan seperti berikut ini.

Di atas, kita tidak memberikan parameter setelah URL /user, sehingga nilai default-nya yang akan ditampilkan, yaitu John.

Sekarang, jika kita berikan parameter setelah /user, maka hasilnya akan seperti berikut ini.

Route Name

Route Name biasanya digunakan untuk mempermudah kita dalam pemanggilan sebuah route saat membangun aplikasi. Jadi kita cukup memanggil name dari route tersebut.

Berikut ini contoh penulisan route name di Laravel. 

Route::get('/user/profile', function () {
    //
})->name('profile');

Di atas, untuk memanggil route kita cukup menggunakan profile dibandingkan harus menuliskan /user/profile.

INFORMASI ! : Route Name bersifat unik dan tidak boleh ada yang sama.

Route Group

Route Group digunakan untuk mengelompokan route-route yang memiliki kondisi yang sama. Misalnya ada beberapa route yang membutuhkan middleware, maka kita bisa menggunakan fitur ini.

Berikut contoh route tanpa Route Group. 

Route::get('/user', [UserController, 'index'])->middleware('auth');
Route::get('/post', [PostController, 'index'])->middleware('auth');
Route::get('/event', [EventController, 'index'])->middleware('auth');

Dari penulisan route di atas, kita bisa gunakan fitur Route Group agar penulisan middleware tidak berulang-ulang. Contohnya seperti berikut ini. 

Route::middleware('auth')->group(function () {

    Route::get('/user', [UserController, 'index']);
    Route::get('/post', [PostController, 'index']);
    Route::get('/event', [EventController, 'index']);
    
});

Route Prefix

Route Prefix bisa digunakan untuk menambahkan awalan URL. Contohnya kita ingin menambahkan awalan admin pada setiap URL route.

Berikut ini contoh penggunaan Route Prefix.

Route::prefix('admin')->group(function () {

    Route::get('/user', [UserController, 'index']);
    Route::get('/post', [PostController, 'index']);
    Route::get('/event', [EventController, 'index']);

});

Jika dijalankan, maka route-route di atas otomatis ditambahkan awalan admin pada awal URL-nya. Sehingga hasilnya akan menjadi seperti berikut ini. 

http://localhost:8000/admin/user

http://localhost:8000/admin/post

http://localhost:8000/admin/event

Penerapan Manual Resource (CRUD)

Seringkali kita perlu membuat route lengkap untuk proses CRUD (Create, Read, Update, Delete). Dibandingkan menggunakan satu fungsi otomatis, menulisnya secara manual memberikan kontrol penuh atas keamanan dan struktur URL kita.

Berikut adalah contoh penerapan Manual Routing yang digabungkan dengan Prefix dan Name untuk fitur manajemen siswa: 

Route::prefix('siswa')->name('siswa.')->group(function () {
    
    Route::get('/', [SiswaController::class, 'index'])->name('index');          // Daftar Siswa
    Route::get('/tambah', [SiswaController::class, 'create'])->name('create');    // Form Tambah
    Route::post('/simpan', [SiswaController::class, 'store'])->name('store');     // Proses Simpan
    Route::get('/{id}/edit', [SiswaController::class, 'edit'])->name('edit');     // Form Edit
    Route::put('/{id}/update', [SiswaController::class, 'update'])->name('update'); // Proses Update
    Route::delete('/{id}/hapus', [SiswaController::class, 'destroy'])->name('destroy'); // Proses Hapus

});

Untuk memanggil route manual tersebut di dalam template Blade (HTML), kita bisa menggunakan potongan kode berikut ini: 

<!-- Tombol Edit -->
<a href="{{ route('siswa.edit', $s->id) }}">Edit</a>

<!-- Form Hapus (Wajib Method DELETE) -->
<form action="{{ route('siswa.destroy', $s->id) }}" method="POST">
    @csrf
    @method('DELETE')
    <button type="submit">Hapus Data</button>
</form>

PENTING ! : Saat menggunakan method PUT atau DELETE di dalam form HTML, kita wajib menambahkan helper @method() karena browser secara default hanya mendukung GET dan POST.
