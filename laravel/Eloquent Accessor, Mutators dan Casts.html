Eloquent Accessor, Mutators dan Casts

  • Mendefinisikan sebuah Accessor
  • Mendefinisikan sebuah Mutator
  • Mendefinisikan sebuah Casting

Laravel memiliki fitur yang bernama Mutator, Accessor dan Casting, fitur-fitur ini digunakan untuk melakukan manipulasi data di dalam attribute database dengan sangat mudah.

Contoh sederhanya misalnya kita ingin melakukan insert data dengan enkripsi ke dalam database dan melakukan deskripsi saat menampilkan dari database secara otomatis, maka kita bisa menggunakan fitur ini.

Mendefinisikan sebuah Accessor

Accessor digunakan untuk melakukan manipulasi nilai attribute saat diakses, untuk mendefinisikan sebuah accessor kurang lebih seperti berikut ini :

<?php
 
namespace App\Models;
 
use Illuminate\Database\Eloquent\Casts\Attribute;
use Illuminate\Database\Eloquent\Model;
 
class User extends Model
{
    /**
     * Get the user's first name.
     *
     * @return \Illuminate\Database\Eloquent\Casts\Attribute
     */
    protected function name(): Attribute
    {
        return Attribute::make(
            get: fn ($value) => strtoupper($value);,
        );
    }
}

Di atas, pertama kita harus melakukan import Elqouent Cast Attribute terlebih dahulu.

use Illuminate\Database\Eloquent\Casts\Attribute;

Setelah itu, kita membuat method baru dengan nama name. Nama method tersebut sesuai dengan nama attribute yang ada di dalam table.

Misalkan nama attribute-nya first_name, maka untuk nama method-nya kita harus menggunakan camelCase. Kurang lebih jadi seperti berikut ini :

protected function firstName(): Attribute
{
	//...
}

Jika kita asumsikan mempunyai data user dengan attribute name dan value Fika Ridaul Maulayya, dan jika kita coba panggil kurang lebih seperti berikut ini :

//import Model "User"
use App\Models\User;

//find user by ID "1"
$user = User::find(1);

//show name user
$name = $user->name;

Jika di jalankan, maka akan menghasilkan seperti ini : FIKA RIDAUL MAULAYYA. Karena di dalam method firstName() kita melakukan return menggunakan strtoupper($value), yang artinya huruf-nya akan di format menjadi kapital.

Mendefinisikan sebuah Mutator

Mutator digunakan untuk melakukan manipulasi nilai attribute saat di set, untuk mendefinisikan mutator kurang lebih seperti berikut ini :

<?php
 
namespace App\Models;
 
use Illuminate\Database\Eloquent\Casts\Attribute;
use Illuminate\Database\Eloquent\Model;
 
class User extends Model
{
    /**
     * Interact with the user's first name.
     *
     * @param  string  $value
     * @return \Illuminate\Database\Eloquent\Casts\Attribute
     */
    protected function name(): Attribute
    {
        return Attribute::make(
            set: fn ($value) => strtolower($value),
        );
    }
}

Setiap kita melakukan insert data name ke dalam database, maka akan di format menjadi huruf kecil semuanya. Karena di dalam method Mutator kita set menggunakan strtolower($value). Yang artinya semua huruf akan di format menjadi kecil.

Mendefinisikan sebuah Casting

Attribute Casting hampir memiliki fungsi yang sama dengan accessor dan mutator, tapi kita tidak perlu melakukan definisi sebuah method di dalam Model. Disini kita hanya mendefinisikan sebuah properti dengan nama $cast dan bisa melakukan konversi dari sebuah attribute ke dalam tipe data umum, seperti :

  • array
  • boolean
  • collection
  • date
  • datetime
  • decimal:<digits>
  • double
  • encrypted
  • encrypted:array
  • encrypted:collection
  • encrypted:object
  • float
  • integer
  • object
  • real
  • string
  • timestamp

Dalam kasus ini kita ambil contoh untuk melakukan konversi attribute email_verified_at yang memiliki tipe data timestap di database, kemudian kita ubah menjadi datetime . Kurang lebih seperti berikut ini :

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class User extends Model
{
    /**
     * The attributes that should be cast to native types.
     *
     * @var array
     */
    protected $casts = [
        'email_verified_at' => 'datetime',
    ];
}
