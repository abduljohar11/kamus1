Views

Tentu saja tidak mungkin melakukan return sebuah HTML di dalam route ataupun controller secara langsung. Untungnya views menyediakan cara yang sangat mudah untuk menempatkan semua file HTML kita secara terpisah.

Dengan views kita bisa memisahkan antara logika aplikasi dengan tampilan aplikasi, sehingga kita akan lebih mudah dalam mengelola HTML untuk ditampilkan pada layar browser. Semua file views akan ditempatkan pada folder resources/views.

Contoh Penggunaan

Contoh sederhana penggunaan views untuk menampilkan data pada route: 

Route::get('/', function () {
    return view('home', ['name' => 'Fika Ridaul Maulayya']);
});

Di atas, kita membuat route dengan path / dan di dalamnya kita melakukan return ke dalam view. Terdapat 2 parameter di sana:

  • Parameter pertama: home (nama file view).
  • Parameter kedua: name => 'Fika Ridaul Maulayya' (data yang dikirim).

Setelah itu, buat file view di resources/views/home.blade.php: 

<html>
    <body>
        <h1>Hello, {{ $name }}</h1>
    </body>
</html>

Jika dijalankan, maka akan mencetak pada layar browser dengan tulisan Hello, Fika Ridaul Maulayya.
