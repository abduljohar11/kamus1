Apa itu Relasi One to One ?

One to One relationship terjadi jika data di dalam table A hanya boleh memiliki satu data di dalam table B (A has one B) dan data B hanya boleh dimiliki oleh data di table A (B Belongs to A).

Dalam kasus ini, kita cukup memberikan sebuah foreign key di dalam table B, dimana foreign key ini akan diisi dengan ID yang ada di dalam table A. Contohnya seperti berikut ini :

table users

id name
1 Fika Ridaul Maulayya
2 Rizqi Maulana
3 Yudi Purwanto

table phones

id phone user_id
1 085785852224 2
2 081200046719 1
3 089223918038 3

field user_id di dalam table phones akan menyimpan nilai id dari table users, dengan begitu maka relasi one-to-one bisa terpenuhi.

Menyiapkan Table users dan phones Menggunakan Migration.

Berikut ini schema untuk table users.

public function up(): void
{
    Schema::create('users', function (Blueprint $table) {
        $table->id();
        $table->string('name');
        $table->string('email')->unique();
        $table->string('password');
    });
}

Di atas kita membuat schema untuk table users dan di dalamnya tidak ada hal yang sepesial, karena yang perlu kita konfigurasi ada di dalam table phones dengan menambahkan foreign key.

public function up(): void
{
    Schema::create('phones', function (Blueprint $table) {
        $table->id();
        $table->string('phone');
        $table->foreignId('user_id')->references('id')->on('users')->unique();
    });
}

Pada file migration di atas, kita menambahkan field user_id dengan mereferensikan id yang ada di dalam table users. Dan foreign key user_id harus bersifat unik atau tidak boleh ada data yang sama.

Mendefinisikan Method Relasi di Model User dan Phone

Agar mudah dalam menggunakan relasi di dalam Laravel, maka kita perlu mendefinisikan method relasi pada kedua model, yaitu User dan Phone.

Untuk model User kita harus menambahkan kode seperti berikut ini.

public function phone()
{
    return $this->hasOne(Phone::class);
}

Kode di atas artinya kita memberi tahu Laravel bahwa kita akan membuat relasi HasOne dari model User ke model Phone atau table phones.

Setelah itu, kita juga perlu menambahkan method relasi di dalam model Phone.

public function user()
{
    return $this->belongsTo(User::class);
}

Kode di atas fungsinya untuk memberitahu Laravel bahwa model Phone ini dimiliki oleh User.

Sehingga kita bisa memanggil data phone yang dimiliki oleh user menggunakan kode seperti berikut ini.

//mencari data user dengan ID "1"
$user = User::find(1);

//tampilkan nomor HP user
$user->phone;

Atau kita juga bisa mencari data user berdasarkan nomor, kurang lebih seperti ini ini.

//cari data nomor dengan "085785852224"
$phone = Phone::where('nomor', 085785852224)->first();

//tampilkan data user
$phone->user;
