Props

Pada artikel kali ini, kita akan membahas tentang Props. Konsep ini sangat penting karena dengan props kita bisa mengirim data dari satu component ke component lain secara dinamis.

Apa Itu Props?

Props (Properties) adalah cara untuk mengirim data dari Parent Component ke Child Component di React. Props bersifat read-only (unidirectional), artinya data yang dikirim tidak boleh diubah langsung oleh child component yang menerimanya.

Jika diibaratkan, props adalah paket yang dikirim dari orang tua ke anak. Anak hanya bisa menggunakan paket tersebut, bukan mengganti label pengirim atau isi asal dari paketnya.

Menggunakan Props di Component

Mari kita buat sebuah component sederhana bernama Welcome yang menerima data nama.

function Welcome(props) {
  return <h1>Halo, {props.name}!</h1>;
}

function App() {
  return (
    <div>
      <Welcome name="Budi" />
      <Welcome name="Siti" />
      <Welcome name="Andi" />
    </div>
  );
}

export default App;

Pada contoh di atas:

  • Kita mengirim atribut name ke component Welcome.
  • React mengumpulkan semua atribut tersebut ke dalam satu objek bernama props.
  • Component anak mengaksesnya menggunakan {props.name}.

Menggunakan Beberapa Props

Kita bisa mengirim banyak data sekaligus melalui props, tidak hanya teks, tapi juga angka, boolean, bahkan fungsi.

function UserProfile(props) {
  return (
    <div>
      <h2>Nama: {props.name}</h2>
      <p>Umur: {props.age}</p>
      <p>Status: {props.isVip ? "Member VIP" : "Regular"}</p>
    </div>
  );
}

function App() {
  return (
    <UserProfile name="Riza" age={25} isVip={true} />
  );
}

Destructuring Props

Supaya kode lebih bersih, para developer React biasanya menggunakan teknik destructuring agar tidak perlu menulis props. berulang kali.

// Langsung ambil property dari parameter
function Welcome({ name, age }) {
  return <h1>Halo {name}, umur kamu {age} tahun.</h1>;
}

Props Bersifat Read-Only

Ingat, jangan pernah mencoba mengubah nilai props di dalam child component seperti ini:

function Welcome(props) {
  props.name = "Ubah Nama"; // ‚ùå ERROR! Jangan lakukan ini.
  return <h1>{props.name}</h1>;
}

Jika kamu butuh data yang bisa berubah-ubah di dalam component, maka kamu harus menggunakan State.

Kesimpulan

Pada artikel ini, kita sudah belajar tentang Props sebagai jembatan komunikasi antar komponen. Kita sudah mencoba mengirim data tunggal, data majemuk, hingga menggunakan destructuring untuk membuat kode lebih elegan.
