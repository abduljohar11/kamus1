Context API

Pada artikel kali ini, kita akan membahas tentang Context API, solusi cerdas untuk berbagi data antar component tanpa terjebak dalam labirin props yang rumit.

ðŸ’¡ Analogi

Bayangkan sebuah Gedung Apartemen. Tanpa Context (Props Drilling), kurir harus menitipkan paket dari resepsionis, ke manajer lantai, baru sampai ke penghuni (Props lewat banyak tangan). Dengan Context API, gedung memiliki Saluran Udara Pusat (Global Store); setiap penghuni cukup memasang masker di kamarnya untuk menghirup udara yang sama secara langsung.

Apa Itu Context API?

Context API adalah fitur bawaan React untuk mengirim data ke banyak component tanpa harus melakukan props drilling. Props drilling terjadi saat kita meneruskan data melalui komponen perantara yang sebenarnya tidak butuh data tersebut.

Langkah 1 - Membuat Context

Pertama, kita buat file khusus (misal UserContext.js) untuk menampung context kita.

import { createContext } from 'react';

// 1. Buat Context
export const UserContext = createContext();

Langkah 2 - Membungkus dengan Provider

Provider bertugas "menyiarkan" data ke seluruh komponen di bawahnya.

import { UserContext } from './UserContext';

function App() {
  const user = { name: "Santri Koding", role: "Admin" };

  return (
    <UserContext.Provider value={user}>
      <Dashboard /> 
    </UserContext.Provider>
  );
}

Langkah 3 - Mengonsumsi Data dengan useContext

Di komponen manapun (misal Profile.jsx) yang berada di dalam lingkup Provider, kita bisa mengambil datanya secara langsung.

import { useContext } from 'react';
import { UserContext } from './UserContext';

function Profile() {
  // 3. Ambil data langsung dari udara (Context)
  const user = useContext(UserContext);

  return <h1>Nama User: {user.name}</h1>;
}

Langkah 4 - State Global (Update Context)

Kita bisa mengirim state dan fungsinya sekaligus agar komponen anak bisa mengubah data global tersebut.

function App() {
  const [theme, setTheme] = useState("light");

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Toolbar />
    </ThemeContext.Provider>
  );
}

// Di komponen Toolbar
const { theme, setTheme } = useContext(ThemeContext);
return <button onClick={() => setTheme("dark")}>Ubah Tema</button>;

Kesimpulan

Context API adalah penyelamat saat aplikasi mulai tumbuh besar. Gunakan fitur ini untuk data yang memang bersifat global seperti tema aplikasi, status login user, atau preferensi bahasa.
