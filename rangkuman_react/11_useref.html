useRef

Pada artikel kali ini, kita akan membahas tentang useRef. Hook ini sering digunakan untuk menyimpan nilai yang "persisten" namun tidak menyebabkan re-render, serta menjadi gerbang utama untuk berinteraksi langsung dengan elemen DOM.

ðŸ’¡ Analogi

Bayangkan sebuah Kotak Penyimpanan Rahasia di balik dinding. Anda bisa memasukkan barang ke dalamnya atau menggantinya kapan saja tanpa harus mendekorasi ulang seluruh ruangan (re-render). Ruangan tetap terlihat sama, tapi isi kotak tersebut tetap ada dan bisa Anda ambil kapan pun dibutuhkan.

Apa Itu useRef?

useRef adalah hook bawaan React yang mengembalikan sebuah objek dengan satu properti saja, yaitu .current. Hook ini berguna untuk:

  • Menyimpan nilai yang bisa diubah tanpa memicu re-render aplikasi.
  • Mengakses elemen DOM secara langsung (seperti document.getElementById).

useRef untuk Mengakses DOM

Contoh paling umum adalah memberikan fokus otomatis pada input saat sebuah tombol diklik.

import { useRef } from "react";

function FocusInput() {
  const inputRef = useRef(null);

  const handleFocus = () => {
    // Mengakses elemen DOM asli melalui properti .current
    inputRef.current.focus();
    inputRef.current.style.backgroundColor = "#fefce8";
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Klik tombol..." />
      <button onClick={handleFocus}>Fokuskan Input</button>
    </div>
  );
}

useRef vs useState

Kapan harus menggunakan salah satunya? Berikut perbandingannya:

Fitur useState useRef
Memicu Re-render? Ya (Setiap perubahan) Tidak (Nilai berubah diam-diam)
Tujuan Utama Data UI (Tampilan) Data Internal / Akses DOM
Sifat Nilai Asinkron Sinkron (Langsung berubah)

useRef untuk Menyimpan Nilai Tanpa Re-render

Gunakan ini jika Anda butuh variabel yang datanya tetap tersimpan meskipun komponen di-render ulang berkali-kali.

function RenderCounter() {
  const [count, setCount] = useState(0);
  const renderCount = useRef(0);

  useEffect(() => {
    // Bertambah tiap render, tapi tidak memicu render tambahan
    renderCount.current = renderCount.current + 1;
  });

  return (
    <div>
      <p>State Count: {count}</p>
      <p>Komponen ini sudah dirender: {renderCount.current} kali</p>
      <button onClick={() => setCount(count + 1)}>Tambah Count</button>
    </div>
  );
}

useRef untuk Menyimpan Data Sebelumnya

Kita bisa memantau perubahan state dengan menyimpan nilai lama ke dalam ref sebelum state tersebut diperbarui.

function PreviousValue() {
  const [text, setText] = useState("");
  const prevText = useRef("");

  useEffect(() => {
    prevText.current = text; // Simpan nilai saat ini ke ref
  }, [text]);

  return (
    <div>
      <input value={text} onChange={(e) => setText(e.target.value)} />
      <p>Sekarang: {text}</p>
      <p>Sebelumnya: {prevText.current}</p>
    </div>
  );
}

Kesimpulan

Pada artikel ini, kita sudah belajar tentang useRef. Gunakan hook ini jika Anda ingin berinteraksi dengan elemen DOM atau menyimpan data yang tidak berpengaruh langsung pada tampilan UI agar performa aplikasi lebih optimal.
