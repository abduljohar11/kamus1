Memoization

Pada artikel kali ini, kita akan membahas tentang Memoization di React, yaitu teknik tingkat lanjut untuk meningkatkan performa aplikasi dengan mencegah render ulang yang tidak perlu.

ðŸ’¡ Analogi

Bayangkan Anda memesan Nasi Goreng yang sama setiap hari. Daripada koki harus memotong bawang dan menggoreng dari nol setiap kali Anda bertanya (re-render), koki menyimpan satu porsi yang sudah jadi di penghangat (cache). Selama pesanan Anda tidak berubah, porsi tersebut langsung diberikan secara instan.

Apa Itu Memoization?

Memoization adalah cara mengoptimalkan aplikasi dengan menyimpan hasil render atau perhitungan. Di React, ada 3 pilar utama:

  • React.memo untuk komponen.
  • useMemo untuk nilai/perhitungan berat.
  • useCallback untuk fungsi agar referensinya tetap.

1. React.memo (Optimasi Komponen)

Mencegah komponen anak me-render ulang jika props yang diterima tidak berubah.

import React from 'react';

const Child = React.memo(({ name }) => {
  console.log("Child dirender...");
  return <p>Halo, {name}!</p>;
});

export default Child;

2. useMemo (Optimasi Nilai)

Menyimpan hasil perhitungan matematika atau manipulasi data yang berat agar tidak dihitung ulang setiap kali render.

import { useMemo, useState } from 'react';

function App() {
  const [count, setCount] = useState(0);

  // Perhitungan ini hanya berjalan jika 'count' berubah
  const total = useMemo(() => {
    console.log("Menghitung ulang...");
    return count * 1000;
  }, [count]);

  return <h1>Total: {total}</h1>;
}

3. useCallback (Optimasi Fungsi)

Penting digunakan saat mengirimkan fungsi sebagai props ke komponen yang dibungkus React.memo. Tanpa ini, fungsi akan dianggap "baru" setiap kali render, sehingga memo komponen anak menjadi sia-sia.

import { useCallback, useState } from 'react';

function App() {
  const [count, setCount] = useState(0);

  // Menjaga referensi fungsi tetap sama
  const increment = useCallback(() => {
    setCount((prev) => prev + 1);
  }, []);

  return <ChildComponent onUpdate={increment} />;
}

Kesimpulan

Gunakan teknik ini dengan bijak. Jangan menggunakannya di semua tempat karena proses memoization sendiri memiliki biaya memori. Gunakan hanya jika komponen terasa lambat atau melibatkan perhitungan data yang sangat besar.
