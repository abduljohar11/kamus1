useEffect

Pada artikel kali ini, kita akan membahas salah satu hook paling sakti di React, yaitu useEffect. Hook ini digunakan untuk menangani efek samping (side effects) yang terjadi di luar kendali alur render utama React.

ðŸ’¡ Analogi

Bayangkan sebuah Toko Kelontong. Alur "Render" adalah saat pemilik menata barang di rak. Side Effect adalah kegiatan tambahan seperti: memesan stok ke supplier saat barang habis (Fetch API), menyalakan lampu saat hari mulai gelap (Update Title), atau mematikan AC saat toko tutup (Cleanup).

Apa Itu useEffect?

useEffect memungkinkan kita menjalankan kode pada siklus tertentu dalam sebuah komponen:

  • Mounting: Saat komponen pertama kali muncul di layar.
  • Updating: Saat ada state atau props yang berubah.
  • Unmounting: Saat komponen dihapus dari layar.

Sintaks Dasar useEffect

useEffect(() => {
  // Kode side effect di sini
}, [dependencies]);

1. Tanpa Dependencies (Berjalan Tiap Render)

Jika kita tidak menyertakan array dependencies, kode di dalamnya akan berjalan setiap kali komponen dirender ulang.

useEffect(() => {
  console.log("Saya berjalan di SETIAP render");
});

2. Array Kosong [] (Hanya Sekali)

Efek ini hanya akan dijalankan satu kali setelah render pertama. Sangat cocok untuk memanggil data dari API.

useEffect(() => {
  console.log("Saya hanya berjalan SEKALI (saat mount)");
  // Contoh: fetchUserData();
}, []);

3. Dengan Dependencies Tertentu

Efek hanya berjalan jika nilai di dalam array dependencies berubah.

const [count, setCount] = useState(0);

useEffect(() => {
  document.title = `Count: ${count}`;
  console.log("Berjalan karena 'count' berubah");
}, [count]); // HANYA berjalan jika count berubah

4. Membersihkan Efek (Cleanup)

Beberapa side effect butuh dibereskan agar tidak terjadi kebocoran memori (memory leak), seperti timer atau subscription.

useEffect(() => {
  const timer = setInterval(() => {
    console.log("Detak jantung aplikasi...");
  }, 1000);

  // Fungsi Cleanup
  return () => {
    clearInterval(timer);
    console.log("Timer dihentikan karena komponen dihapus");
  };
}, []);

Kesimpulan

useEffect adalah jantung dari interaksi eksternal di React. Dengan memahami penggunaan array dependencies, kita bisa mengontrol kapan kode kita harus berjalan secara efisien dan mencegah bug yang tidak diinginkan.

Di artikel berikutnya, kita akan membahas tentang useRef, hook yang memungkinkan kita menyimpan nilai yang persisten tanpa memicu re-render.
