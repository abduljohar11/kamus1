Form (Controlled vs Uncontrolled)

Pada artikel kali ini, kita akan membahas tentang Form di React. Memahami cara kerja form sangat krusial karena hampir setiap aplikasi memerlukan input dari pengguna, mulai dari pencarian sederhana hingga pendaftaran akun yang kompleks.

ðŸ’¡ Analogi

Bayangkan sebuah Mobil. Controlled Component seperti mobil modern dengan layar digital; setiap kali Anda memutar setir sedikit saja, sensor mendeteksi dan memperbarui angka di layar (state). Uncontrolled Component seperti mobil tua tanpa indikator; Anda bebas memutar setir, dan Anda baru tahu posisinya saat Anda menoleh keluar jendela untuk mengecek roda (direct DOM access).

Apa Itu Form di React?

Di React, ada dua cara utama untuk mengelola data dari elemen input seperti <input>, <textarea>, dan <select>:

  • Controlled Component: React memegang kendali penuh atas nilai input melalui state.
  • Uncontrolled Component: Browser atau DOM memegang nilai tersebut, dan kita hanya mengambilnya saat dibutuhkan menggunakan ref.

Controlled Component

Inilah cara yang paling direkomendasikan. Nilai input selalu sinkron dengan state React.

import { useState } from "react";

function ControlledForm() {
  const [name, setName] = useState("");

  const handleSubmit = (e) => {
    e.preventDefault();
    alert("Nama yang dikirim: " + name);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input 
        type="text" 
        value={name} 
        onChange={(e) => setName(e.target.value)} 
      />
      <button type="submit">Kirim</button>
    </form>
  );
}

Uncontrolled Component

Cara ini menggunakan useRef untuk menarik nilai dari DOM saat dibutuhkan (biasanya saat submit).

import { useRef } from "react";

function UncontrolledForm() {
  const inputRef = useRef(null);

  const handleSubmit = (e) => {
    e.preventDefault();
    alert("Nama dari DOM: " + inputRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" ref={inputRef} />
      <button type="submit">Kirim</button>
    </form>
  );
}

Form dengan Beberapa Input

Untuk form yang memiliki banyak field, kita bisa menggunakan satu state object agar kode tetap ringkas.

function MultiInputForm() {
  const [form, setForm] = useState({ email: "", password: "" });

  const handleChange = (e) => {
    setForm({
      ...form,
      [e.target.name]: e.target.value
    });
  };

  return (
    <form>
      <input name="email" onChange={handleChange} placeholder="Email" />
      <input name="password" type="password" onChange={handleChange} />
    </form>
  );
}

Kesimpulan

Kita telah belajar bahwa Controlled Component adalah standar industri karena memberikan kendali penuh untuk validasi dan manipulasi data. Namun, Uncontrolled Component bisa menjadi solusi cepat jika Anda hanya butuh mengambil satu nilai saat submit tanpa perlu re-render setiap ketikan.
