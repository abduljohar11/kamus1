Edit & Update Data

üìù Analogi: Proses Edit adalah seperti mengambil berkas lama dari rak (database) untuk diletakkan di meja kerja (form). Sedangkan Update adalah proses menghapus tulisan lama dan menggantinya dengan informasi baru, lalu mengembalikannya kembali ke dalam rak.

Langkah 1 - Menambahkan Method Edit & Update

Buka app/Http/Controllers/PostController.php dan tambahkan method berikut untuk menangani pengambilan data lama dan pemrosesan perubahan.

public function edit(Post $post)
{
    // Mengambil data post dan mengirimnya ke view Edit.jsx
    return inertia('Posts/Edit', [
        'post' => $post,
    ]);
}

public function update(Request $request, Post $post)
{
    // 1. Validasi input
    $request->validate([
        'title'   => 'required',
        'content' => 'required',
    ]);

    // 2. Update data di database
    $post->update([
        'title'   => $request->title,
        'content' => $request->content
    ]);

    // 3. Redirect ke index dengan pesan sukses
    return redirect()->route('posts.index')->with('message', 'Data Berhasil Diupdate!');
}

Langkah 2 - Menambahkan Tombol Edit di Index

Buka resources/js/Pages/Posts/Index.jsx dan tambahkan tombol Edit di dalam kolom aksi pada tabel:

<td className="p-3 text-center">
    <Link href={`/posts/${post.id}/edit`} className="bg-blue-500 text-white px-3 py-1 rounded shadow-sm hover:bg-blue-700">
        EDIT
    </Link>
</td>

Langkah 3 - Membuat View Edit Data

Buat file baru resources/js/Pages/Posts/Edit.jsx. Kita akan mengisi state awal dengan data props.post yang dikirim dari controller.

import React, { useState } from 'react';
import Layout from '../../Layouts/Default';
import { router } from '@inertiajs/react';

export default function EditPost({ post, errors }) {
    // State dengan default value dari props post
    const [title, setTitle] = useState(post.title);
    const [content, setContent] = useState(post.content);

    const updatePost = (e) => {
        e.preventDefault();
        
        router.put(`/posts/${post.id}`, {
            title: title,
            content: content
        });
    }

    return (
        <Layout>
            <div className="bg-white p-6 rounded shadow-md">
                <form onSubmit={updatePost}>
                    <div className="mb-4">
                        <label className="block font-bold">TITLE</label>
                        <input type="text" className="w-full border p-2 rounded" value={title} onChange={(e) => setTitle(e.target.value)} />
                        {errors.title && <div className="text-red-500 mt-2">{errors.title}</div>}
                    </div>
                    <div className="mb-4">
                        <label className="block font-bold">CONTENT</label>
                        <textarea className="w-full border p-2 rounded" rows="5" value={content} onChange={(e) => setContent(e.target.value)}></textarea>
                        {errors.content && <div className="text-red-500 mt-2">{errors.content}</div>}
                    </div>
                    <button type="submit" className="bg-blue-600 text-white px-4 py-2 rounded">UPDATE</button>
                </form>
            </div>
        </Layout>
    );
}

Kesimpulan

Berkat fitur Route Model Binding pada Laravel, kita bisa langsung mendapatkan instance data hanya melalui ID di URL. Di materi terakhir, kita akan menyelesaikan CRUD ini dengan fitur Hapus Data (Delete).
